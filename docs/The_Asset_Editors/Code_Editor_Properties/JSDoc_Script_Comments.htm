JSDoc スクリプトコメント

カスタム関数のコード補完を有効にし、引数などをコードエディタで表示したい場合は、JSDoc 形式コメントを追加しましょう。このコメントを追加すると、スクリプトエディタで関数をどのように扱い、どのように入力するかが自動補完によって認識されます。

関数のヘッダーとして一般的な形式は、関数の名前、説明、そして関数が受け取る引数（パラメーター）を並べるというもので、JSDoc 形式のコメントとして解析されるように各行は " ///" で始めなくてはなりません。それぞれのコメントにも識別子（先頭に "@" を付ける）と内容を指定する必要があります。利用できる識別子は以下のとおりです。

 

識別子 内容
@function / @func 関数の名前と引数です。例:
my_func(x, y, colour)
@description / @desc 関数の動作についての概要です。
@param / @arg / @argument 引数の型（任意）は{}で囲み、あとは引数の名前、簡単な説明となります（それぞれ半角スペースで区切ります）。

 

関数を書くにあたってどう機能するのかを掴むためにも、基本的な例を挙げましょう。

function is_same_object(_id, _obj)
{
if _id.object_index == _obj
    {
    return true;
    }
else return false;
}

このスクリプトは指定したオブジェクトと同じ object_index がインスタンスにあるかをチェックするためのもので、このように簡単に呼び出せます。

if is_same_object(id, obj_Player)
    {
    instance_destroy()
    }

とはいえ、このように書いてもコードエディタには引数も、使い方も表示されません。よって説明と名前、引数を JSDoc コメントとして追加しなくてはなりません。

/// @function            is_same_object(id, object)
/// @description         Compare an instance object index with that of another.
/// @param {real} _id    The unique instance ID value of the instance to check.
/// @param {real} _obj   The object index to be checked against.
function is_same_object(_id, _obj)
{
if _id.object_index == _obj
    {
    return true;
    }
else return false;
}

こうしてコードから関数を呼び出すと、自動補完されて引数と説明などが表示されるようになります。

画像の上側にあるのは関数の自動補完、下側にあるのは引数ヘルパーの動作です。なお、@param における任意の「型」と必須の「説明」の両方は、デフォルトでは IDE によるコード補完で表示されず、確認するには GML 環境設定のオプションを有効にしなくてはなりません。
