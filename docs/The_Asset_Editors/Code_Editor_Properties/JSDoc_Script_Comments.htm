JSDoc スクリプトコメント

カスタム関数のコード補完を有効にし、引数などをコードエディタで表示したいなら、JSDoc 形式コメントを追加しなくてはなりません。これらのコメントを追加すると、自動補完によってスクリプトエディタでの関数の使い方、記入方法が記録されます。

関数のヘッダーとして一般的な形式は、関数の名前、説明、さらに関数が受け取る引数（パラメーター）を並べるというもので、JSDoc 形式のコメントとして解析させるには行の先頭に " ///" を加えなくてはなりません。コメントそれぞれにも識別子（ "@" で始まる）、内容を指定する必要があります。利用できる識別子は以下のとおりです。

 

識別子 内容
@function / @func 関数の名前と引数であり、以下のような形式となります。
my_func(x, y, colour)
@description / @desc 関数の動作についての全体的な説明です。
@param / @arg / @argument {}で囲まれた引数の型（任意）、引数の名前、要約となります（それぞれ半角スペースで区切られます）。

 

関数を書くにあたってどう機能するのか分かるように、基本的な例を挙げましょう。

function is_same_object(_id, _obj)
{
if _id.object_index == _obj
    {
    return true;
    }
else return false;
}

このスクリプトは指定したオブジェクトと同じ object_index がインスタンスにあるかをチェックするためのもので、このように簡単に呼び出せます。

if is_same_object(id, obj_Player)
    {
    instance_destroy()
    }

とはいえコードエディタに関数を書いても、どのような引数があるのか、どのように使えばいいのかは表示されません。よって説明と名前、引数を JSDoc コメントとして追加しなくてはなりません。

/// @function            is_same_object(id, object)
/// @description         Compare an instance object index with that of another.
/// @param {real} _id    The unique instance ID value of the instance to check.
/// @param {real} _obj   The object index to be checked against.
function is_same_object(_id, _obj)
{
if _id.object_index == _obj
    {
    return true;
    }
else return false;
}

こうしてコードから関数を呼び出すと、自動補完によって引数と説明といったものが表示されるようになります。

ウィンドウの上側に示されているのは関数の自動補完、下側に示されているのは引数ヘルパーです。なお、@param 内にある任意の「型」、必須の「説明」のいずれも、デフォルトでは IDE によるコード補完で表示されないため、確認するには GML 環境設定の各オプションを有効にする必要があります。
