コードの編集

スクリプトエディタを使い始めるのは簡単で、スクリプトアセットやイベントなどに追加したいコードを入力すればいいのですが、GML の正しい構文を守らなくてはなりません。入力中は自動補完ウィンドウを参照できます。

このウィンドウはコードを入力しているときに表示され（少し時間を置いてから開き、環境設定からその時間を変更できます）、入力内容のもっとも近い候補がアルファベット順に表示されます。

コードエディタでは、ドラッグでコードの行を、を押しながらドラッグでコードの列を選択できます。選択した内容は / + " C" でコピー、 / + "X" で切り取り、 / + "V" で貼り付けが可能です。

コードにはさまざまな形でコメントを入れることができます。効率的に進めるにはコメントを入れて内容を分かりやすくし、それぞれの部分で行われている処理、役割をはっきりさせることが大切です。チームで開発している場合は、他のメンバーにメモを残すことで自分が行った変更や、コードの各部分がもつ役割を正しく伝えることができます。個人開発でもコメントを残すことは大切で、開発からいったん離れ、後になって再開するような場合には大いに役立つでしょう。

コメントの入れ方は簡単で、行の先頭に "//" を書き加えるだけです。あるいは "/* テキスト内容 */" というように書くと、囲んだ部分全体をコメントアウトすることができます。キーボードショートカットも利用可能で、 / + "K" で選択行をコメント化、 / + + "K" でコメント解除できます。画像のコードにはこの二通りの方法でコメントが入れられており、さらには JS Doc コメントも含まれています（詳しくは以下）。

スクリプトには JSDoc 形式のコメントを追加し、スクリプト内にある関数のプロパティを定義することもできます。そうした関数は自動補完によって引数、説明とともに表示されるようになります。詳しくは JSDoc スクリプトコメントを参照してください。

デフォルトでは、波括弧 "{}" の横にはコードの折りたたみアイコンが表示されます。このアイコンは+または-の形をしており、クリックすると括弧内のコードが折りたたまれ、見通しが良くなります。

コードの折りたたみ領域は、#region 用のタグによって各部分を区切ることで手動設定できます。手順としては、#region タグを各部分の先頭に（このタグの奥にはコメントを追記できます）、#endregion タグを最後に入れることで折りたたみが可能となります。折りたたまれたコードは非表示となるため、このように表示行を減らすことができます。

実際のコードは以下のようになります。

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

ご存知かもしれませんが、テキストの各部分は色分け表示されます。スクリプトエディタでは既存のオブジェクト、組み込み変数、関数といったものが認識され、それぞれに合わせて色分けしてくれるのです。コードが色分けされることで、名前を間違って入力したり、キーワードを変数として扱ってしまったり、構文エラーが生じてしまった場合にもぱっと見て分かるようになります。必要なければ環境設定から無効にできるほか（エディタ内からでも F10 で切り替え可能です）、色を変更することが可能です。

コードエディタを右クリックすると、次のメニューが表示されます。

このメニューにはコード行の切り取り、コピー、貼り付けといった基本機能に加えて、選択テキストのコメント / コメント解除、インデントの追加 / 削除、さらにコードウィンドウを最大 3 つのペイン（区画）に分離できる選択項目があります（なお#regions 用の項目についてはコード内に折りたたみ領域がないと表示されません）。

ペインに分離するとウィンドウが複数の領域に分かれます。それぞれスクリプトまたはイベントをドラッグして追加でき、隣のペインと比較しながら編集を行うことができます。

特に有用となるのは 2 つのスクリプトを比較したり、繋ぎ合わされているスクリプトを同時に確認するといった場面で、新しいウィンドウを開いたり、タブを切り替えるといった手間を省けます。

あとの選択項目は、デバッガのブレークポイントを切り替えるための項目、GML のコードを DnD™ に（または DnD™ のコードを GML に）変換するための項目となります。なお、コードを DnD™ に変換した場合、コードの各行からアクションを作成するのではなく、1 つの Code アクションにイベントまたはスクリプトの内容をすべて収めます。
