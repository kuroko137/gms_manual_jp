コードの編集

スクリプトエディタを使い始めるのは簡単で、スクリプトアセットやイベントなどに追加したいコードを入力するだけですが、GML の正しい構文どおりに入力する必要があります。同時にこのような自動補完ウィンドウが開きます。

このウィンドウはコードの入力中に表示され（少し時間を置いてから開き、環境設定からその時間を変更できます）、入力内容のもっとも近い候補がアルファベット順に表示されます。

コードエディタでは、ドラッグでコードの行を、を押しながらドラッグでコードの列を選択できます。選択した内容は / + " C" でコピー、 / + "X" で切り取り、 / + "V" で貼り付けられます。

コードにはさまざまな形でコメントを入れることができます。効率的に進めるにはコメントを入れて内容を示すことが大事で、それぞれの部分で行われている処理、そして役割がはっきりと分かるようになります。また、チームで開発しているなら他のメンバーに向けたメモを含めることで、自分が行った変更や、コードのそれぞれの部分の役割を正しく伝えることができます。個人開発でもコメントを残すことは大事です。特に開発からいったん離れ、後になって再開するようなことがあれば大いに助かるでしょう。

コメントの入れ方は簡単で、行の先頭に "//" を書き加えるだけです。あるいは "/* テキスト内容 */" というように書けば、囲んだ部分全体をコメントアウトすることができます。キーボードショートカットもあり、 / + "K" で選択行をコメント化、 / + + "K" でコメント解除できます。画像のコードにはこれら 2 つの方法によってコメントが入れられており、さらに JS Doc コメントも含まれています（詳しくは以下で解説）。

スクリプトには JSDoc 形式のコメントを追加し、スクリプト内にある関数のプロパティを定義することもできます。このようにして関数を呼び出すと、引数および説明がオートコンプリートで表示されます。詳細については JSDoc スクリプトコメントを参照してください。

デフォルトでは、波括弧（{}）に収まる部分にはコードの折りたたみアイコンが表示されます。このアイコンは+または-の形をしており、クリックすると括弧内のコードが折りたたまれ、表示部分がコンパクトになります。

また、#region という専用のタグでそれぞれの部分を示すことで、コードの折りたたみ領域を手動で設定できます。使い方としては、まず#region タグ（続けてコメントにしたいテキストを入力可）をそれぞれの部分の先頭に加え、それから#endregion タグを最後に加えればコードを折りたためるようになります。折りたたまれたコードは非表示となるため、エディタに表示するコードの行数を少なくすることができます。

実際にはこのような見た目となります。

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

お気付きでしょうが、テキストの各部分は色分け表示されます。スクリプトエディタでは既存のオブジェクト、組み込み変数、関数といったものがすべて認識されており、それぞれに合わせて色分けしてくれるのです。コードが色分けされることで、名前を間違って入力したり、キーワードを変数として扱ってしまったり、構文エラーを起こしてしまった場合にもすぐ分かるようになります。色分け表示をしたくない場合でも、環境設定から無効にしたり（エディタ内からでも F10 で切り替えできます）、色を変更することが可能です。

エディタ内を右クリックすると、このようなメニューが表示されます。

メニューからはコード行の切り取り、コピー、貼り付けといった基本機能のほかに、選択テキストのコメント / コメント解除、インデントの追加 / 削除、さらにコードウィンドウを 3 つまでのペインに分離できます（なお#regions に関する選択項目についてはコード内で事前に定義されていないと表示されません）

複数のペインに分離するとウィンドウが複数の領域に分かれます。それぞれにスクリプトやイベントをドラッグして追加でき、それらを比較しながら編集できます。

2 つのスクリプトを比較したり、連結されているスクリプトを同時に確認するといった場面で役立ち、新しいウィンドウを開いたり、あるいはタブを常に切り替える必要もありません。

他の選択項目を選ぶとデバッガのブレークポイントを切り替えたり、GML のコードを DnD™ に（または DnD™ のコードを GML に）変換することができます。なお、コードを DnD™ に変換した場合、コードの各行からアクションを作成するのではなく、1 つの Code アクションにイベントおよびスクリプトの内容をすべて収めます。
