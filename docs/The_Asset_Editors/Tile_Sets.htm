タイルセットエディタ

タイルセットとはレベル（ステージ）や、それ以外の動かない要素を描画するためのグラフィックリソースです。タイルセットはセル（各タイル）ごとに分けられた 1 枚の画像からなり、それぞれのタイルをルームエディタで配置して新しい画像を作り上げることになります。以下はタイルセットとして扱えるスプライトの例です。

画像提供: Lost Garden

タイルセットを作る上でもっとも一般的な形式は、タイルをすき間なく詰め込むか、少しの余白を空けるというもので、GameMaker Studio 2 ではこの両方を扱うことができます。基本的にあらゆるタイルセットは明確に定義された、各セルが同じ大きさのグリッドに収まる画像で作られています（なお上記の画像ではすべてのタイルが真四角ですが、タイルセットは長方形のタイルでも作成できます）。

タイルセットを作成する際は、一番左上のグリッドセルが必ず空になることを忘れないでください。このタイルは GameMaker Studio 2 では「空のタイル」として扱われ、既存のタイルを消すためのものとなります（そのため画像の左上部分にあるピクセルは無視されます）。

タイルセットリソースを作成すると、タイルセットエディタのウィンドウが開いて以下の項目が表示されます。

 

名前名前

タイルセットに名前を付けることができます。英数字および "_" 記号のみ利用可能で、先頭の文字は数字以外にしなくてはなりません。


スプライトスプライト

タイルセットを構成するスプライトアセットを選択できます。大きいボタンをクリックするとアセットエクスプローラーが開きますので、タイルセットに利用するスプライトを指定しましょう。

また、スプライト新規作成ボタンをクリックすると新しいスプライトアセットを作成できます（ワークスペースにスプライトエディタも開かれます）。スプライト編集ボタンをクリックした場合はスプライトを編集でき、ボタンの場合はスプライトの画像をイメージエディタで編集できます。

タイルセットを開いた際、デフォルトではタイルセットプロパティのウィンドウも表示されます。このウィンドウは閉じてしまっても、ボタンをクリックして開き直すことができます。


プロパティプロパティ

タイルセットプロパティのウィンドウでは、タイルの基本サイズ、オフセット量、タイル間の余白を指定できます（最後の 2 つはページ上部のサンプル画像のような、各部分にすき間があるタイルセット用のオプションです）。また、出力境界の値を設定するとタイルセット画像をディスク上にどう保存するかを制御し、必要に応じて各タイルの周囲にピクセルを追加できます。

タイルのレンダリング方法にまつわる理由で、タイルには縁を「塗りつぶす」ための境界をあらかじめ用意しなくてはなりません。以下の画像は、デバイスのグラフィックドライバーによってタイルが通常どのようにマッピングされるかを示したもので、タイルの周りにある赤線がマッピングされる U,V 座標となります（ "U" と "V" は 2D テクスチャメッシュを言い表す文字で、これはすでに X 、Y 、Z がルームとオブジェクトの空間座標を表す文字として使われているためです）。

ルームに画像を描画する際は、テクセルの中心を選んで直接マッピングを行います。これが等倍なら何も問題ありませんが、タイルを縮小している場合はどうでしょう？ここで問題となるのは、縮小されるのは画像だけでなく UV も同じで、UV はより狭くなった空間に押し込まれるものの、決して収まらないということです。そのため利用可能なピクセル中心に基づき、グラフィックドライバーによって描画するテクセルが選択されます。

そして新しい画面ピクセルに移動しかかっている場合、グラフィックドライバーによってピクセルの中心から描画内容が決められることになり、タイル上のテクセルではなく、タイルの周りのテクセルが選択されます。

上記の画像は、6 つの画面ピクセルに縮小描画されたタイルで、隣の画面ピクセルに少しだけ重なっている場合、グラフィックドライバーによってテクセルを切り捨てるか、有効と見なしたテクセルによって塗りつぶすか選ばれることを示しています。この場合、重なってしまったテクセルを画面ピクセルに描画することになりますが、見てのとおりタイルの外に出ているため、隣のタイル、または指定したタイル周辺のわずかな空間からピクセルが取得されます。つまりタイルを縮小すると「異常」な描画によってタイルの間に亀裂が入ることがあり、場合によっては多くの乱れが生じてしまいます。グラフィックドライバーとデバイスはそれぞれ独自の方法で描画を行うため、予想のつかない悪い見た目となる場合がありますが、あらかじめ気をつけて対処することは可能です。

ハイエンドゲームで 1 枚のテクスチャ（テクスチャページおよびテクスチャアトラス上にないもの）から描画する場合、クランプというテクスチャモードを使用するオプションがあります。このオプションはテクセルの最終行を無限に繰り返すというもので、ハードウェアによって強制的に最終行のピクセルが取得されるようになるため、問題なくスケーリングできるようになります。これこそが出力境界設定の働きです。

画像に表示されているのは「固定」されたタイルスプライトで、見てのとおりタイルの周辺部分が繰り返されています。これはハードウェア側での超過が起きた場合、直後にある空間かタイルのいずれかではなく、必ずタイルの最終テクセルが選択されるということを意味します。ですから、ビューのズーム操作をしたり、タイルを拡大縮小してレンダリングするつもりなら、タイルセットを作成するときに出力境界の縦横幅をデフォルト値の 2 より大きくする必要があることを忘れないでください。

下のオプションからは、タイルセットリソースの帰属先となるテクスチャグループを指定できます（あらかじめテクスチャグループマネージャーから作成する必要があります）。テクスチャグループとは、特定のルームあるいはレベルに使われる画像アセットをまとめて保存するためのもので、レベル 1 用の画像をあるグループに、レベル 2 用の画像を別のグループに入れるといったように設定します。こうしてまとめられたアセットは同じテクスチャページに置かれ、指定したターゲットプラットフォームでの実行時、テクスチャページのスワップ（切り替え）を減らすことができます。

メモ: テクスチャページの設定は必須ではなく、パフォーマンスを向上できるかは対象デバイスが CPU 、GPU のどちらに依存しているかによって異なります。

一番下にあるのは、元スプライトのエクスポートを無効にするオプションです。最初からチェックされており、プロジェクトのテクスチャページが生成される際に元のスプライトアセットが除外されるようになります。これはスプライトがタイルセットとして扱われており、他の場所では使用されないという条件下でのみ有効となり、エクスポートを一度で済ませることができます。タイル以外にも使われている場合はチェックを外しておき、コンパイル時に元スプライト、タイルセットの両方がテクスチャページに追加されるようにしましょう。

エディタエディタ

エディタの領域にあるボタンをクリックすると、タイルセット用のエディタを機能別に開くことができます。エディタからは、複数のタイルを組み合わせたタイルセットブラシ、フレームごとにタイルが変化するアニメーションタイル、周囲のタイルに合わせて組み替えられるオートタイルを作成できます。各エディタの詳しい内容については、ページの一番下のリンク先を参照してください。


プレビューツールボックスプレビューツールボックス

タイルセットに指定したスプライトを確認するためのツールボックスです。それぞれのボタンをクリックしてズーム倍率を変更でき、ボタンをクリックした場合はプレビューを等倍表示に、ウィンドウにフィットボタンをクリックした場合は、タイルセット全体が表示されるようエディタの画面を合わせることができます（必要に応じて拡大縮小されます）。ズーム倍率は / を押しながらマウスホイールでも変更可能で、 / + で等倍にリセットすれば実際のサイズを確認できます。

また、ボタンをクリックした場合はグリッドの表示が切り替わるほか、ボタンをクリックして色を変更できます。グリッドを有効にすると枠内のタイルに色つきの線が表示され、分割などの設定も反映されます。


プレビュープレビュー

タイルセットのプレビューウィンドウです。通常のワークスペースと同じように基本的な操作でナビゲート可能で、中央マウスボタン（または + ）でドラッグするとキャンバスの移動、マウスホイールで垂直スクロール、 / キーとマウスホイールでズームインとズームアウトができます。また、プレビューツールボックスのボタンをクリックするとズーム倍率を変更したり、セルの上に表示されるグリッドを切り替えることができます。

 

エディタの節に書かれているように、タイルセットを編集して専用のブラシ（ルームに描くための複合タイル）、アニメーションタイル、オートタイル（ルームに置かれると自動でつなぎ合わされる特別なタイル）を生成するツールが用意されており、それぞれ以下のページで詳しく解説されています。

  • ブラシビルダー
  • アニメーションタイル
  • オートタイル
