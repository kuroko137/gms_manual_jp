タイルセットエディタ

タイルセットとはレベル（ステージ）や、それ以外の動かない要素を描画するためのグラフィックリソースです。タイルセットはセル（タイル）ごとに分けられた 1 枚の画像からなり、それぞれのタイルをルームエディタで配置して新しい画像を作り上げることになります。以下はタイルセットとして扱えるスプライトの例です。

画像提供: Lost Garden

タイルセットを用意する上でもっとも一般的な手法は、すべてのタイルを詰め込むか、少しの余白を空けるという手法で、GameMaker Studio 2 ではこの両方を扱うことができます。基本的にあらゆるタイルセットは明確に定義された、各セルが同じ大きさのグリッドに収まる画像で作られています（なお上記の画像ではすべてのタイルが真四角ですが、タイルセットは長方形のタイルでも作成できます）。

タイルセットを作成する際は、一番左上のグリッドセルが必ず空になることを忘れないでください。このタイルは GameMaker Studio 2 では「空のタイル」として扱われ、既存のタイルを消すためのものとなります（そのため画像の左上部分にあるピクセルは無視されます）。

タイルセットリソースを作成すると、タイルセットエディタのウィンドウが開いて以下の項目が表示されます。

 

名前名前

タイルセットに名前を付けることができます。英数字および "_" 記号のみを利用でき、先頭の文字は数字以外にしなくてはなりません。


スプライトスプライト

タイルセットを構成するスプライトアセットを選択できます。大きいボタンをクリックするとアセットエクスプローラーが開きますので、タイルセットに利用するスプライトを指定しましょう。

また、スプライト新規作成ボタンをクリックすると新しいスプライトアセットを作成できます（ワークスペースにスプライトエディタも開かれます）。スプライト編集ボタンをクリックした場合はスプライトを編集でき、ボタンの場合はスプライトの画像をイメージエディタで編集できます。

タイルセットを開いた際、デフォルトではタイルセットプロパティのウィンドウも表示されます。このウィンドウは閉じてしまっても、ボタンをクリックして開き直すことができます。


プロパティプロパティ

タイルセットプロパティのウィンドウでは、タイルの基本サイズ、オフセット量、タイル間の余白を指定できます（最後の 2 つはページ上部のサンプル画像のような、各部分にすき間があるタイルセット用のオプションです）。また、出力境界の値を設定するとタイルセット画像をディスク上にどう保存するかを制御し、必要に応じて各タイルの周囲にピクセルを追加できます。

タイルのレンダリング方法に対応するため、タイルには周りに引き伸ばされる（塗りつけられる）縁をあらかじめ用意しなくてはなりません。以下の画像は、デバイスのグラフィックドライバーによって通常タイルがどのようにマッピングされるかを示したもので、タイルの周りにある赤線がマッピングされる U,V 座標となります（ "U" と "V" は 2D テクスチャメッシュを表すための文字で、これは X 、Y 、Z がすでにルームとオブジェクトの空間座標を表しているためです）。

ルームに画像を描画する際は、テクセルの中心を選んで直接マッピングを行います。これが等倍なら何も問題ありませんが、タイルを縮小している場合はどうでしょう？何が問題かというと、縮小されるのは画像だけでなく UV も同じで、UV はより狭くなった空間に押し込まれるものの、決して収まらないということです。そのため利用可能なピクセル中心に基づき、グラフィックドライバーによって描画するテクセルが選択されます。

そして新しい画面ピクセルに移動しかかっている場合、グラフィックドライバーによってピクセルの中心から描画内容が決められることになり、タイル上のテクセルではなく、タイルの周りのテクセルが選択されます。

上記の画像は、6 画面ピクセルに縮小描画されたタイルで、隣の画面ピクセルに少しだけ重なっているとき、グラフィックドライバーによってそのテクセルを削除するか、有効と見なしたテクセルによって塗りつぶすか選ばれることを示しています。この場合、重なり合ったテクセルを画面ピクセルに描画することになりますが、明らかにタイルの外にはみ出てしまっているため、隣のタイル、またはタイルの周りに用意したわずかな空間からピクセルが取得されます。よってタイルを縮小すると、描画方法における「不具合」によってタイルの間に亀裂が生じ、多くの乱れが生じてしまうでしょう。グラフィックドライバーとデバイスはそれぞれ独自の方法で描画を行うため、結果が予測できずにひどい見た目になる場合がありますが、注意して計画的に行えば対処できます。

ハイエンドゲームで 1 枚のテクスチャ（テクスチャページおよびテクスチャアトラス上にないもの）から描画する場合、クランプというテクスチャモードを使用するオプションがあります。このオプションはテクセルの最終行を無限に繰り返すというもので、ハードウェアによって強制的に最終行のピクセルが取得されるようになるため、問題なくスケーリングできるようになります。以下は出力境界設定の効果となります。

画像に表示されているのは「固定」されたタイルスプライトであり、見てのとおりタイルの周辺部分が繰り返されています。これはハードウェアでの超過が起きた場合、いずれかの空間や直後のタイルではなく、必ずタイルの最終テクセルが選択されるということを意味します。そのためタイルセットを作成するにあたり、ゲームビューをズーム操作したり、タイルを拡大縮小してレンダリングするつもりなら、出力境界の縦横幅をデフォルト値の 2 より大きくしなければならない場合があることを覚えておいてください。

下にある項目からは、タイルセットリソースを入れるテクスチャグループを指定できます。テクスチャグループとは、特定のルームやレベルといったものに使われる画像アセットをまとめて保存するために用意するものです（あらかじめテクスチャグループマネージャーから定義しておきます）。たとえば、レベル 1 に含まれている画像をあるグループに、レベル 2 に含まれている画像を別のグループに入れるといったように利用でき、このようにしてまとめられたアセットは同じテクスチャページに置かれ、指定したターゲットプラットフォームで実行しているときのテクスチャページのスワップ（切り替え）を減らすことができます。

メモ: テクスチャページの設定は必須ではなく、パフォーマンスを向上できるかは対象デバイスが CPU 、GPU のどちらに依存しているかによって異なります。

一番下にあるのは、元スプライトのエクスポートを無効にするオプションです。最初からチェックされており、この状態ではプロジェクトのテクスチャページが生成される際に元のスプライトアセットが除外されるようになります。これはタイルセットとして扱われているスプライトで、他の場所では使われないという前提の下で機能しており、エクスポートが一度で済むようになります。タイル以外にも使われている場合はチェックを外しましょう。そうするとコンパイル時、元スプライトとタイルセットが両方ともテクスチャページに追加されます。

エディタエディタ

エディタの部分にある各ボタンをクリックすると、機能別にタイルセット用のエディタが開きます。複数のタイルからなるタイルセットブラシ、フレームごとにタイルが変わるアニメーションタイルのほかに、周囲のタイルに応じて繋ぎ合わされるオートタイルを作成できます。それぞれのエディタについては、ページの一番下のリンク先で詳しく説明されています。


プレビューツールボックスプレビューツールボックス

タイルセットに指定したスプライトを確認するためのツールです。ボタンを押してプレビューの表示を拡大縮小でき、ボタンをクリックするとプレビューを等倍表示にリセットでき、ウィンドウにフィットボタンをクリックした場合は、タイルセット全体が表示されるようエディタの画面をフィットさせます（必要に応じて拡大縮小されます）。ズーム倍率は / を押しながらマウスホイールでも変更可能で、 / + で等倍にリセットすれば実際のサイズを確認できます。

ほかにもボタンをクリックしてグリッドの表示を切り替えたり、ボタンをクリックしてグリッドの色を変更することが可能です。グリッドを有効にすると、グリッド内のタイルに色つきの線が表示され、これには分割などの設定も反映されます。


プレビュープレビュー

この部分はタイルセットのプレビューウィンドウです。通常のワークスペースと同じように基本的な操作でナビゲート可能で、中央マウスボタン（または + ）でドラッグするとキャンバスの移動、マウスホイールで垂直スクロール、 / キーとマウスホイールでズームインとズームアウトができます。また、プレビューツールボックスのボタンをクリックするとズーム倍率を変更したり、セルの上に表示されるグリッドを切り替えることができます。

 

エディタの節で話したように、他にもタイルセットを編集したり、専用のブラシ（ルームに描画できる組み合わせタイル）、アニメーションタイル、オートタイル（ルームに置かれると自動でつなぎ合わされる特別なタイル）を生成するためのツールがあります。これらは以下のページで詳しく掘り下げられています。

  • ブラシビルダー
  • アニメーションタイル
  • オートタイル
