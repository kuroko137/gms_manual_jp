タイルセットエディタ

タイルセットとはレベル（ステージ）や、それ以外の動かない要素を描画するためのグラフィックリソースです。タイルセットはセル（タイル）ごとに分けられた 1 枚の画像からなり、それぞれのタイルをルームエディタで配置して新しい画像を作り上げることになります。以下はタイルセットとして扱えるスプライトの例です。

画像提供: Lost Garden

タイルセットを用意する上でもっとも一般的な手法は、すべてのタイルを詰め込むか、少しの余白を空けるという手法で、GameMaker Studio 2 ではこの両方を扱うことができます。基本的にあらゆるタイルセットは明確に定義された、各セルが同じ大きさのグリッドに収まる画像で作られています（なお上記の画像ではすべてのタイルが真四角ですが、タイルセットは長方形のタイルでも作成できます）。

タイルセットを作成する際は、一番左上のグリッドセルが必ず空になることを忘れないでください。このタイルは GameMaker Studio 2 では「空のタイル」として扱われ、既存のタイルを消すためのものとなります（そのため画像の左上部分にあるピクセルは無視されます）。

タイルセットリソースを作成すると、タイルセットエディタのウィンドウが開いて以下の項目が表示されます。

 

名前名前

タイルセットに名前を付けることができます。英数字および "_" 記号のみを利用でき、先頭の文字は数字以外にしなくてはなりません。


スプライトスプライト

タイルセットを構成するスプライトアセットを選択できます。大きいボタンをクリックするとアセットエクスプローラーが開きますので、タイルセットに利用するスプライトを指定しましょう。

また、スプライト新規作成ボタンをクリックすると新しいスプライトアセットを作成できます（ワークスペースにスプライトエディタも開かれます）。スプライト編集ボタンをクリックした場合はスプライトを編集でき、ボタンの場合はスプライトの画像をイメージエディタで編集できます。

タイルセットを開いた際、デフォルトではタイルセットプロパティのウィンドウも表示されます。このウィンドウは閉じてしまっても、ボタンをクリックして開き直すことができます。


プロパティプロパティ

タイルセットプロパティのウィンドウでは、タイルの基本サイズ、オフセット量、タイル間の余白を指定できます（最後の 2 つはページ上部のサンプル画像のような、各部分にすき間があるタイルセット用のオプションです）。また、出力境界の値を設定するとタイルセット画像をディスク上にどう保存するかを制御し、必要に応じて各タイルの周囲にピクセルを追加できます。

タイルのレンダリング方法に対応するため、タイルには周りに引き伸ばされる（塗りつけられる）縁をあらかじめ用意しなくてはなりません。以下の画像は、デバイスのグラフィックドライバーによって通常タイルがどのようにマッピングされるかを示したもので、タイルの周りにある赤線がマッピングされる U,V 座標となります（ "U" と "V" は 2D テクスチャメッシュを表すための文字で、これは X 、Y 、Z がすでにルームとオブジェクトの空間座標を表しているためです）。

ルームに画像を描画する際は、テクセルの中心を選んで直接マッピングを行います。これが等倍なら何も問題ありませんが、タイルを縮小している場合はどうでしょう？何が問題かというと、縮小されるのは画像だけでなく UV も同じで、UV はより狭くなった空間に押し込まれるものの、決して収まらないということです。そのため利用可能なピクセル中心に基づき、グラフィックドライバーによって描画するテクセルが選択されます。

そして新しい画面ピクセルに移動しかかっている場合、グラフィックドライバーによってピクセルの中心から描画内容が決められることになり、タイル上のテクセルではなく、タイルの周りのテクセルが選択されます。

上記の画像は、6 画面ピクセルに縮小描画されたタイルで、隣の画面ピクセルに少しだけ重なっているとき、グラフィックドライバーによってそのテクセルを削除するか、有効と見なしたテクセルによって塗りつぶすか選ばれることを示しています。この場合、重なり合ったテクセルを画面ピクセルに描画することになりますが、明らかにタイルの外にはみ出てしまっているため、隣のタイル、またはタイルの周りに用意したわずかな空間からピクセルが取得されます。よってタイルを縮小すると、描画方法における「不具合」によってタイルの間に亀裂が生じ、多くの乱れが生じてしまうでしょう。グラフィックドライバーとデバイスはそれぞれ独自の方法で描画を行うため、結果が予測できずにひどい見た目になる場合がありますが、注意して計画的に行えば対処できます。

ハイエンドゲームで 1 枚のテクスチャ（テクスチャページおよびテクスチャアトラス上にないもの）から描画する場合、クランプというテクスチャモードを使用するオプションがあります。このオプションはテクセルの最終行を無限に繰り返すというもので、ハードウェアによって強制的に最終行のピクセルが取得されるようになるため、問題なくスケーリングできるようになります。以下は出力境界設定の効果となります。

画像に表示されているのは「固定」されたタイルスプライトであり、見てのとおりタイルの周辺部分が繰り返されています。これはハードウェアでの超過が起きた場合、いずれかの空間や直後のタイルではなく、必ずタイルの最終テクセルが選択されるということを意味します。そのためタイルセットを作成するにあたり、ゲームビューをズーム操作したり、タイルを拡大縮小してレンダリングするつもりなら、出力境界の縦横幅をデフォルト値の 2 より大きくしなければならない場合があることを覚えておいてください。

下にある項目からは、タイルセットリソースを入れるテクスチャグループを指定できます。テクスチャグループとは、特定のルームやレベルといったものに使われる画像アセットをまとめて保存するために用意するものです（あらかじめテクスチャグループマネージャーから定義しておきます）。たとえば、レベル 1 に含まれている画像をあるグループに、レベル 2 に含まれている画像を別のグループに入れるといったように利用でき、このようにしてまとめられたアセットは同じテクスチャページに置かれ、指定したターゲットプラットフォームで実行しているときのテクスチャページのスワップ（切り替え）を減らすことができます。

メモ: テクスチャページの設定は必須ではなく、パフォーマンスを向上できるかは対象デバイスが CPU 、GPU のどちらに依存しているかによって異なります。

The final option in the tile set properties is Disable Source Sprite Export. This is checked by default, and what it does is flags the source sprite asset to be omitted when the texture page for your project is being generated. This works under the assumption that a sprite used as a tile set will not be getting used for anything else, and so only needs to be exported once. However if you are using the base sprite resource for anything other than tiles, then you would want to un-check this so that the base sprite and the tile set are both added into the texture page on compile.

EditorsEditors

Each of the buttons in the Editors section will open a different editor for specific tile set functionality. You can create tile set brushes based on multiple tiles together, animated tiles, using different tiles as individual frames, and you can also create auto tiles that will "join" together based on the tiles that surround them. Each of these editors is explained in detail in the sections linked at the end of this page.


Preview ToolboxPreview Toolbox

These are the tools you can use to preview the sprite selected for the tile set. These buttons control the current preview zoom level and you can use them to zoom in or out and clicking the button will reset the preview to be 1:1 with the tile set sprite being edited. You can also click the Window Fit button to make the entire tile set fit within the current preview window (this will zoom in/out as appropriate to make it fit). Note that you can also zoom in and out using the / and the Mouse Wheel , and pressing / + will set the canvas to be 1:1 with the tile set sprite.

Another feature is that you can switch the grid view on or off by clicking the button, as well as set the colour for the grid to be used (clicking the button). When the grid is active it will show a coloured outline for each of the tiles in the grid, respecting the settings for separation, etc...


PreviewPreview

This section is the tile set Preview window. To navigate around the preview, you can use the same basic controls as for general Workspaces, ie: Use the middle mouse button and drag to pan the preview (or alternatively use the + combination), and you can scroll horizontally with the mouse wheel or zoom in and out using the / key and the mouse wheel . You can also use the different buttons in the Preview Toolbox to change the zoom and enable/disable the grid cell overlay.

 

As mentioned in the section on Editors above, there are are further tools available to edit tile sets and generate special brushes (combinations of tiles that can be painted into a room), animated tiles, and auto tile tiles (which are special tiles that will connect up automatically when placed in a room).
