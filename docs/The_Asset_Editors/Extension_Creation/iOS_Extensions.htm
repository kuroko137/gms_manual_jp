iOS / tvOS 拡張機能

iOS および tvOS の拡張機能は 2 つの手順から作成します。1 つ目は拡張機能および必要ファイル等の追加、2 つ目は関数およびマクロ / 定数の作成です。関数と定数はプレースホルダによってグループ化して追加されるため、こちらで説明されているようにプレースホルダのファイルを追加してから関数とマクロを定義しなくてはなりません。残りのファイルを追加するには、まずエディタの追加プラットフォーム領域にある iOS および（または）tvOS をチェックし、拡張機能プロパティのウィンドウを開いてください（下の画像は iOS のプロパティウィンドウですが、tvOS のウィンドウもまったく同じものです）。

以下はプロパティの詳細です。

  • リンカーフラグ / コンパイラフラグ: 一部のフレームワークやサードパーティ SDK を動作させるには、リンカーフラグおよびコンパイラーフラグをこちらに入力する必要があります（詳しくは SDK またはフレームワークの同梱ドキュメントを参照してください）。
  • クラス名: 拡張機能には複数のクラスを与えることができ、クラスごとに専用の関数、定数を用意できます。用途がわかるような名前にしましょう。
  • アプリデリゲートクラス名: アプリのカスタムデリゲートクラス名です。この設定をすると、拡張機能によってコアアプリの機能をオーバーライド / 拡張できるようになります。利用するには以下のようにする必要があります。
    1. 各デリゲートソースファイルの名前をデリゲートクラスの名前に合わせます（例: @interface MyCustomAppDelegate を "MyCustomAppDelegate.h" で定義）
    2. アプリのデリゲートヘッダーファイル先頭に環境変数${YYExtAppDelegateIncludes} を追加します。これはコンパイル時、親デリゲートクラスの関連インクルードファイルに置き換えられます。
    3. 環境変数${YYExtAppDelegateBaseClass} をカスタムアプリデリゲートのベースクラスとして使用します。これはコンパイル時、適切なベースデリゲートクラスに置き換えられます。

      カスタムアプリデリゲートを利用している他の拡張機能と併用させるには、子クラスのオーバーライドされたメソッドからベースクラスメソッドを呼び出す必要があります。スーパークラスメソッドを呼び出す前に、エラーを避けるためクラス階層に組み込まれていることを以下のように確認しなければなりません。

    - (BOOL)application:(UIApplication *)application willFinishLaunchingWithOptions:(NSDictionary *)launchOptions
    {
        // Check if any superclasses implement this method and call it
        if([[self superclass] instancesRespondToSelector:@selector(application:willFinishLaunchingWithOptions:)])
          return [super application:application willFinishLaunchingWithOptions:launchOptions];
        else
            return TRUE;
    }

  • システムフレームワーク: iOS のシステムフレームワークを拡張機能に追加できます。ボタンをクリックするとプレースホルダのフレームワークが追加され、ゆっくりダブルクリックすると編集できます。利用可能なフレームワークについてはこちらに詳しく記載されています。フレームワークを削除するには、削除したいフレームワークを選んでボタンをクリックしてください。
  • Info.plist へ挿入: info.plist ファイルへ挿入するコードを追加できます。
  • サードパーティフレームワークとバンドル: ここからはサードパーティおよび SDK バンドルを追加できます。システムフレームワークと同じく、ボタンをクリックして追加、ゆっくりダブルクリックして編集、をクリックして削除となります（フレームワーク名については選択した SDK に含まれているドキュメントを参照してください）。フレームワーク /SDK バンドルが開発用 Mac にある場合、「 Mac のフレームワークパスを入力」にパスを追加してボタンをクリックするか、SDK かソースコードが PC 上にある場合は「ソースを追加」ボタンで指定してください。
  • CocoaPods: 追加された SDK およびフレームワークの代わりに（あるいは同時に）使用する CocoaPods の名前を入力します。iOS と tvOS ではゲームオプションから GameMaker Studio2 に Podfile を追加でき、ここに CocoaPods を追加する場合に必要となります。
  • CocoaPods 依存関係: ここには前の部分で追加した CocoaPods によって必要とされる依存関係をリストアップできます。なお、iOS および tvOS のゲームオプションから Podfile.lock を追加でき、ここに CocoaPods 依存関係を追加する場合に必要となります。

拡張機能にシステムフレームワークあるいはサードパーティフレームワークが追加されている場合、拡張機能プロパティウィンドウにチェックボックス付きでリストアップされます。チェックを入れるとウィークリンクが有効となり、ライブラリに含まれているシンボルを好きな値に「オーバーライド」する上で役に立ちますが、リンクが遅くなるため注意してください。
