ルームの継承

ルームエディタには、継承という重要な機能があります。継承とは、「親」にしたいルームから「子」のルームを作成し、親のプロパティをすべて子に引き継がせることを意味します。この子ルームは親の複製といえるもので、あらゆる要素が親ルームとリンクしています。つまり親ルームのビューを変更したり、タイルを動かしたり、インスタンスを追加するなどした場合、子ルームにもすべて反映されるということです。さらに子ルームでは、レイヤー、インスタンス、アセットといった要素の一つ一つで継承を無効にすることが可能です。

プロパティを引き継いでいるルームでは、画像のように「継承」ボタンが強調表示されます。このボタンをクリックすると、レイヤーやリソースなどの継承を切り替えることができます。

最初はちょっと分かりづらいので、例を出してみましょう。たとえば中央に町とお城が置かれたルームがあるとします。町とお城はそれぞれ別のタイルレイヤーで描画されており、周囲には道が敷かれ、NPC たちが歩いています。この城はゲームの途中で破壊されますが、町の様子は元のままにしたいとしましょう。まずは通常どおりに初期ルームを作成し、それから親ルームのレイヤーを引き継ぐ子ルームを作成します。そして子ルームを開き、城のタイルレイヤー（または個々のタイル）の継承を「オフ」に切り替えたのち、タイルを編集して破壊された城を作りましょう。こうしてプレイヤーが子ルームに入ると、親ルームのインスタンス、タイルといったすべてが同じで、城だけが破壊されていることになります。

重要！ 継承されたプロパティに編集を加えた場合、ボタンをクリックしなくても自動的に継承が無効となります。

ルームを継承させるには、親にしたいルームをアセットブラウザで選択してから右クリックしてください。ルームの右クリックメニューが開きますので、子を作成を選びましょう。新しい子ルームが作成され、アセットブラウザではこのように親ルームに連結して表示されます。

ルームの順番および継承は、アセットブラウザ右上のメニューからルームマネージャーを開くと変更できます。「継承関係」のタブに並んでいるルームは、別のルームにドラッグすることでそのルームの子にできます（それとは逆に、子ルームを親ルームの外にドラッグすると継承を破棄できます）

ルームにおける継承とは多層的なものであり、一つ一つの要素、レイヤー内にまとめられている要素、それらのプロパティ、さらにはルーム全体を継承するようチェックできるほか、ルーム A を継承しているルーム B を、ルーム C が継承するというように、ルームを連続的に継承することも可能です。たとえばビューの設定だけがされている元ルームを作成し、以降のルームをすべて子ルームとした場合、元ルームのビュープロパティがそれぞれの子ルームに引き継がれます。さらに応用編として、「ビュー」のルームを親とした元ルームを新しく作成し、あらゆるルームに存在させたい GUI 用、管理用のオブジェクトを配置してみましょう。こうして子ルームを作成すると、そのルームは親ルームから管理用、GUI 用のオブジェクトを継承し、さらにその親ルームからビューの設定を継承します。

ここまでお読みになられた通り、大いに役立つ機能です。継承によってルームの構造をすっきりさせれば作業も早く進められ、ルームのデザインに費やす時間を最小限にできます。
