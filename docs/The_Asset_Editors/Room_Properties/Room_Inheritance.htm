ルームの継承関係

ルームエディタには、継承という重要な機能があります。継承とは、あるルームを「親」のルームとし、それから親のルームのプロパティをすべて引き継いだ「子」のルームを作成するということを意味します。この子ルームは親ルームの複製といえるもので、あらゆる要素が親ルームとリンクしています。つまり親側のビューを変更したり、タイルを動かしたり、インスタンスを追加するなどした場合、そべてが子ルームにも反映されるということです。また、子ルームでは特定のレイヤー、インスタンス、アセットだけといったように、部分的に継承を無効にすることも可能です。

プロパティを引き継いでいるルームでは、以下の画像のように「継承ボタン」が強調表示されます。このボタンをクリックすると、レイヤーやリソースといった要素の継承を切り替えることができます。

いきなりでは分かりづらいので、ちょっとした例を出しましょう。たとえば中央に町とお城が置かれたルームがあるとします。町とお城はそれぞれ別のタイルレイヤーで描画されており、その周りには道が敷かれ、NPC たちが歩いています。そしてゲームの途中で城が破壊されるものの、町の様子は元のままにしたいとしましょう。まずは通常どおりに初期ルームを作成し、それから親ルームのレイヤーを引き継ぐ子ルームを作成します。それから子ルームを開き、城用のタイルレイヤー（または個々のタイル）の継承を「オフ」に切り替えたのち、タイルを編集して破壊された見た目にしましょう。こうしてプレイヤーが子ルームに入った場合、親ルームのインスタンス、タイルといったすべてが同じものの、城だけが破壊されていることになります。

重要！ 継承されたプロパティに編集を加えた場合、ボタンをクリックしなくても自動的に継承が無効となります。

To create room inheritance, you simply select the room you want to create a child from in the Asset Browser and then right-click . This will open the room menu and there you can select Create Child to create a new child room. This will be created and "attached" to the parent visually in the Asset Browser:

To change the room order and/or inheritance you need to use the Room Manager, which you can open using the menu in the top right of the Asset Browser. Here you can go to the Inheritance tab and then click and drag any room onto any other room to have it become a child of that room (or drag a child room off a parent and break the inheritance).

Note that, within a room, inheritance is multi-layered and you can check individual items, groups of items on a single layer, item properties, or whole rooms as inheriting, and you can also inherit from consecutive rooms (so room C inherits from room B, which in turn has inherited from Room A). If you create a base room with nothing in it except the view setup - for example - you can then create all further rooms as being its children and they will all inherit the view properties of the parent room. But you can go a step further and create another base room with the "view" room as its parent which holds the general GUI instance and the controllers required in every game room. Now you can create child rooms from this and they will inherit both the controllers and GUIs from the parent room as well as the view setup from the parent of that room!

As you can hopefully see, inheritance is very powerful and you can use it to maintain a much cleaner room structure, speed up the workflow and generally optimise the time you spend making and designing rooms.
