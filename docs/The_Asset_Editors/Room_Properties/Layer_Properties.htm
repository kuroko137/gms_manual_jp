レイヤーの種類とプロパティ

ルームエディタで作成したものはすべてレイヤーの中に置かれます。レイヤーはいずれもレイヤーエディタから追加・削除したり、整理することができます。レイヤーにはタイルレイヤー、パスレイヤー、インスタンスレイヤーといった種類があり、それぞれに専用の設定、仕組みがあります。数には上限がなく、どんな種類のレイヤーでも複数追加可能です。レイヤーには種類ごとにプロパティウィンドウがあり、レイヤーの種類によって設定できる内容は異なります。レイヤーは表示の切り替えも可能で、非表示にしたレイヤーは実行時に処理されません（ただし専用のレイヤー関数によっていつでも再表示できます）。

レイヤーエディタはこのような見た目をしています。上部はレイヤーリスト、下部はレイヤー作成ボタンとなります。表示されている名前をゆっくりダブルクリックするとレイヤーの名前を変更でき（英数字および "_" 記号のみ受け付けます）、目アイコンをクリックするとレイヤーの表示を切り替えられます。親ルームから継承されたレイヤーがある場合、下のボタンから継承の切り替えができますが、すべてのレイヤーに影響するためご注意ください（こちらで詳しく解説されています）。なお、または / を押しながらレイヤーをクリックすると複数選択でき、まとめて複製したり削除できるようになります（ボタンだけでなく右クリックメニューからも同じ操作が可能です）。

レイヤーの並びを整理したくなったら、フォルダボタンをクリックしてレイヤーフォルダを作成し、レイヤーをフォルダの中に入れましょう。削除ボタンを押した場合は、選択中のレイヤーをまとめて削除できます。すべてのレイヤーには深度値という、ルームにレイヤーの内容がレンダリングされたときに、そのレイヤーが描画順のどこに置かれるかを決める値があります。レイヤーは高い深度から低い深度へと描画されるため、深度が-100 のレイヤーがある場合、深度が 200 のレイヤーより前に描画されます。

重要！レイヤーの深度には-16000 から 16000 までの上限値があり、上限外のレイヤーに置かれたインスタンスは描画されません。ただしレイヤー上にあるもののイベントはすべて通常どおりに実行されます。

レイヤーを右クリックすると、以下のレイヤーメニューが開きます。

このメニューからはレイヤープロパティウィンドウを開いたり、レイヤーの削除、名前変更、さらにはサブレイヤーを追加できます。サブレイヤーを追加した場合、現在のレイヤーにはサブレイヤーが付属していることを示す「フォルダ」アイコンが与えられ、新しいレイヤーは現在のレイヤーの下、右にタブを空けて作成されます。あとはサブレイヤーに親レイヤーのプロパティを継承させるか選択したり、親レイヤーの表示を継承させるかを設定することができます。

なお、ウィンドウ内のレイヤーは上下にドラッグすると順番を変更できるほか、を押しながらクリックするとあるレイヤーから別のレイヤーまで複数選択、移動できたり（間にあるすべてのレイヤーごと）、 / を押しながらクリックで個別選択もできます。レイヤーをレイヤーフォルダの上にドラッグするとフォルダの中に移動し、そのフォルダのサブレイヤーとなります。また、ロックボタンをクリックするとレイヤーをロックし、間違って編集できないようになります。

メモ: エディタのウィンドウではそれぞれ異なるアセットをレイヤーに配置することになりますが、"P" を押している間はアセットをすぐに選択でき、そのアセットが配置されているレイヤーに切り替えることができます。

レイヤーの種類、およびレイヤーの編集可能プロパティについては以下でそれぞれ説明されています。

 

背景背景

このボタンをクリックすると背景レイヤーが追加され、背景のプロパティウィンドウが開きます。

背景レイヤーを作成すると最初は空となっており、背景色が描画されます。これは背景レイヤーの画像となるスプライトが割り当てられていないためです。なお、色をクリックするとカラーピッカーを開いて背景色を変更できます。スプライトが指定されている場合、水平タイル、垂直タイル、画像をルーム全体にストレッチさせるかの切り替えが可能となります。

背景を決められた方向にスクロールさせたいなら、水平・垂直速度を設定しましょう（ゲームステップごとに一定ピクセル動くようになります）。また、X / Y のオフセット値を設定すれば背景画像の位置を変えることができます。指定したスプライトリソースに複数のサブ画像がある場合、背景のアニメーション速度を設定できますが、デフォルトではスプライトエディタの設定値にロックされます。レイヤーの深度も指定できますが、同じくデフォルトではロックされています。値を変更するだけなら、レイヤーエディタからレイヤーをドラッグして順番を変えるだけで可能です。

ウィンドウの一番上のボタンをクリックした場合、背景の継承が切り替わります。

 

インスタンスインスタンス

このボタンをクリックするとインスタンスレイヤーが追加され、インスタンスのプロパティウィンドウが開きます。

このウィンドウには、レイヤーに追加されているインスタンスのリストが表示されます。アセットブラウザを開いてオブジェクトをルームまでドラッグすると、レイヤーに新しいインスタンスを追加できます。またはアセットブラウザからインスタンスを選択後、を押している間はリソースがその場にプレビュー表示され、そのまま左クリックするとインスタンスを追加、ドラッグすると「描く」ように連続して追加できます。

リストに並んでいるインスタンスはドラッグして自由に並び替えできますが、あくまでルームエディタのリスト表示を整理できるだけであり、インスタンスが生成される順番を変更したい場合は、ルームプロパティのウィンドウ、またはルームの IDE メニューから生成順序ウィンドウを開かなくてはなりません。

ルームエディタまたはプロパティに表示されているインスタンスをダブルクリックすると、新しいウィンドウを開いて個々のインスタンスの性質を変更できます。

インスタンスには "inst_38B0BFF" のようなデフォルト名が与えられますが、この名前は自由に変更できます（英数字および "_" 記号のみ使用でき、スペースは受け付けません）。注意点として、ここで指定した名前は定数として扱われるため（自分でつけた名前か IDE によるデフォルト名かを問わず）、インスタンスごとに固有の名前にしなくてはなりません。この名前はコードからインスタンスを識別するための名前となりますが、識別できるのはインスタンスが同じルームに存在する場合のみです。

インスタンス名の下にある各ボタンからは、インスタンスを親ルームから継承させるかの設定、オブジェクトの変更、オブジェクトエディタを開いて元のオブジェクトを呼び出せるほか、オブジェクト変数の編集と確認ができます。インスタンスが最初からもつ特徴を変えることも可能で、ブレンドされる色（デフォルトでは白色）、回転および反転、縦横のスケール変更、ルーム内の位置、スプライトの画像速度、初期フレームといったものを設定できます。

他に設定できるものは、インスタンスの生成コードです。このコードはオブジェクトの Create イベントの直後に実行され、そのインスタンスだけのものとなります（たとえば元のオブジェクトで設定した変数を、インスタンスの生成コードから独自の値に変更するといったことが可能です）。ボタンをクリックするとスクリプトウィンドウがインスタンスの横に開き、関数を追加したり、変数を変更できるようになります。コードが実行されるタイミングはインスタンスが生成され、Create イベントが実行された直後です。なおインスタンスの継承とは別に、生成コードの継承を切り替えることも可能です。

レイヤー上のインスタンスは、ゲーム作成時に自動でエクスポートするようフラグ付けされます。とはいえ継承を利用していたり、常に存在し続けるデバッグ用オブジェクトが用意されている場合は、完成したゲームのルームに入れたくないインスタンスが見つかるかもしれません。このような場合、リストのインスタンスの横にある "エクスポート" のチェックを外せば、実行ファイルの一部として出力されなくなります。ただし大きな注意点があり、チェックを外したインスタンスをコードから参照している場合、ゲームが正常に実行されなくなる恐れがあります。そのため注意して扱ってください。

 

タイルマップタイルマップ

このボタンをクリックするとタイルマップレイヤーが追加され、タイルマップレイヤーのプロパティウィンドウとタイル編集ウィンドウが開きます。

タイルマップレイヤーのプロパティでは、レイヤーの継承切り替え、使用するタイルセットの変更ができ、レイヤーのグリッドをルームの (0, 0) に揃えたくない場合は X / Y のオフセットを設定できます。レイヤーがレンダリングされる深度を設定することもでき（専用の継承ボタンもあります）、ロックをすれば不要な編集を防ぐことが可能です。

レイヤープロパティのウィンドウを開くとさらにタイル編集ウィンドウも開き、現在のタイルセットが表示されます（ウィンドウを間違って閉じてしまった場合、IDE 上部のメニューからルーム > タイル編集というように選択すれば開き直せます）。クリックで選択したタイルはブラシになり、ルームエディタのキャンバス上をクリックすれば現在のレイヤーに描くことができます。複数のタイルで描きたくなったら、タイルセットのタイルをドラッグするとまとめて選択できます。ブラシに追加したいタイルが離れている場合、 / を押しながら左クリックでタイルを個別に選択し、カスタムブラシにできます（を押しながらクリックでタイルの除外もできます）。こうして作られたカスタムブラシは一時的にしか使えませんが、ブラシタブで作ったものはタイルマップレイヤーに保存されるため無くなりません。 

タイル編集ウィンドウではマウスホイール、上部の各ボタンによってタイルのズーム倍率を変更できるほか、中央ボタンによるドラッグまたはを押しながらドラッグでスクロールできます。このウィンドウにはグリッドオプションもあり、有効にしてタイルを区切ったり、グリッドそのものの色を変更できます。なお、グリッドを非表示にしてもタイルは必ずスナップします。

このウィンドウにはライブラリタブもあり、ブラシ、アニメーションタイル、オートタイルを選択できます。これらに関してはタイルセットエディタのページで取り上げられています。

タイルレイヤーの選択時は、ルームエディタの上部にタイル用のツールが表示されます。以下はその一覧です。

鉛筆 鉛筆ツールを使うと、選択中のタイルをブラシとしてルームエディタに左クリックでタイルを描くことができ、右クリックするとタイルを消せます。ルームエディタのステータスバーにはタイルのインデックス値が表示され、複数のタイルをブラシとして選択している場合、すべてのタイルのインデックスが一覧となって表示されます。
消しゴム 消しゴムツールを使うと、ルームエディタのタイルを左クリックで消すことができます。実際にはタイルのインデックス値を、空のタイルのインデックス値として予約されている 0 に変更しています。
塗つぶし 塗りつぶしツールではタイルマップレイヤー全体、あるいは選択中のタイルを現在のタイルブラシで塗りつぶすことができます。ブラシには単一のタイル、複数のタイルからなる複合ブラシ、さらにはアニメーションライブラリのタイルを利用でき、レイヤー上をクリックすると指定したブラシで塗りつぶされます。選択範囲がない場合は空のタイル、または同じインデックスのタイルがつながっている部分を塗りつぶします。選択範囲がある場合、範囲内のすべてを塗りつぶします。
直線 直線ツールを選択すると、ドラッグしてタイルの直線を描けるようになります。タイルマップレイヤーに選択範囲がある場合、範囲内にのみ直線で描けます。このツールでは 1 つのタイルのブラシ、複数のタイルのブラシ、ライブラリのアニメーションタイルのいずれも利用できます。
長方形 長方形ツールを選択すると、ドラッグしてタイルで塗りつぶした長方形を描けるようになります。タイルマップレイヤーに選択範囲がある場合、範囲内にのみ長方形で描けます。このツールでは 1 つのタイルのブラシ、複数のタイルのブラシ、ライブラリのアニメーションタイルのいずれも利用できます。
選択 選択ツールを使うと、タイルレイヤーの編集範囲を決めることができます。左クリックをしてドラッグすると長方形の選択範囲ができ、 / を押しながらドラッグで選択範囲の追加、を押しながらドラッグで選択範囲の削除ができます。選択範囲をすべて消したいときはを押しましょう。タイルレイヤーに選択範囲がある場合、ほかのツール（鉛筆、反転、回転など）はその選択範囲内でのみ扱うことができます。また、ルームエディタで選択されているタイルはコピー（ / + "C" ）、切り取り（ / + "X" ）、貼り付け（ / + " V" ）も可能で、この場合は鉛筆ツールに切り替わり、選択中のタイルでルームのレイヤーに描くことができます。
オートタイル Clicking this tool enables the Auto tiling paint style. When this is active you can select any tile from the Autotile Libraries tab, and then paint it into the room layer and GameMaker Studio 2 will automatically change it to match the surrounding tiles, as long as you have correctly set up the Auto Tile Tab. Note that selecting a tile from the tile set that is not part of the autotile libraries will reset the drawing tool to the standard Pencil tool.
Inheritance With this tool you can enable/disable inheritance for specific tiles on a given layer. For this to work, the room must first be a child room and the layer must have it's inheritance switched on in the layer properties. When in this state, the room editor will colour all inherited tiles red by default, and clicking the right mouse button will switch off inheritance for that tile, and clicking the left mouse button will switch it back on again. In this way you can re-use elements from "base" rooms, only selectively changing the tiles that you require while inheriting all the rest.
Flip Clicking the Flip tool with the left mouse button (or using the keyboard shortcut "X" ) will flip the tile currently selected for drawing along the horizontal axis, without changing drawing tool (if you have a custom brush selected for drawing, the whole brush will flip). If you have no tile selected for drawing, but do have a group of tiles selected on a layer, then the flip tool will Flip the selected tiles.
Mirror Clicking the Mirror tool with the left mouse button (or using the keyboard shortcut " Y" ) will mirror the tile currently selected for drawing along the vertical axis, without changing drawing tool (if you have a custom brush selected for drawing, the whole brush will mirror). If you have no tile selected for drawing, but do have a group of tiles selected on a layer, then the Mirror tool will mirror the selected tiles.
Rotate Clicking the Rotate tool with the left mouse button (or using the keyboard shortcut " Z" ) will rotate the tile currently selected for drawing 90° clockwise, without changing drawing tool (if you have a custom brush selected for drawing, the whole brush will rotate). If you have no tile selected for drawing, but do have a group of tiles selected on a layer, then the Rotate tool will rotate the selected tiles.

Note that if you use the Selection Tool to select a group of tiles in the room editor canvas and then either cut or copy them, they will temporarily become the current brush for painting.

 

PathsPaths

You can add a new path layer by clicking on the button. This will add the new layer, and then open up the path properties window:

Here you can select the path from the Asset Browser to use for the layer and then change the defined connection type (either using smooth interpolation or straight), as well as whether it is closed or not and the precision with which the path should be created. Note too that you can toggle the inheritance here, and that you can set the path to be a specific colour (default is red) to make seeing them better in the editor. If you have no path resource defined, you can click the New button to create one. Path layers also have a depth value associated with them (locked by default) even though nothing is actually rendered in your game when using them.

It is worth noting that you can also add points to the path by clicking the left mouse button , as well as remove points using the right mouse button , or select a point by clicking on it and dragging it around, all within the actual room editor, making creating and editing paths very easy (although you can still edit the path in the Path Editor as normal too). If you hold down / + on different path points, then they will all be selected and you can move them together or delete them all from the menu option (explained below).

If you right click in the room editor window while manipulating a path then you'll get the following options:

Here you can reverse the path direction (this will have little visible effect, but reverses the order in which the path points are counted so the start point becomes the end point and vice-versa), as well as flip them horizontally or vertically. You can also delete the selected point from this menu. Note that you can hold down "T", " S" or "R" then click and drag to translate, scale or rotate the entire path.

 

AssetsAssets

You can add a new asset layer by clicking on the button. This will add the new layer, and then open up the asset layer properties window:

An asset layer is simply a graphical asset (sprite or sequence) that has been placed into the room independently of instances, and you can place them by either dragging them in from the Asset Browser or by selecting the one you want in the Asset Browser and then using + to "paint" them. Sprite assets are similar to tiles, only they can be fully animated - if the sprite used has sub-images they will be shown - without the restrictions that are in place for tiles, ie: they aren't forced to a grid and animations can be any length and speed. Sequences are "cellections" of sprites, sounds and instances that will play when the room is entered.

While placing assets on the asset layer they will snap to the grid snap values set at the top of the room editor for the grid, but if you want to place them freehand, simply switch off the grid or use the keys / and the left mouse button . While those keys are held down, you will be free to place the asset anywhere, without it snapping to the grid resolution.

Once you place an asset in the room, you have a certain amount of control over how it looks by double clicking on it to open the asset properties window:

From here you can give the asset a unique name, set whether it is to inherit from the parent room, or change the sprite or sequence that you want it to display. For sequeunces, you can set the animation speed and the initial playhead frame, as well as the colour to blend with it (white by default) and the position in the room. You can change the characteristics of sprite assets too, setting a colour to blend it with (white by default), or rotating and flipping it. You can also scale sprites along either or both the axis, and set their position within the room. If the sprite has sub-images, you can choose the animation speed as well as set the initial frame to be shown.

It is worth noting that each asset on the layer is automatically flagged for exporting when the game is made. However, especially when working with inheritance, it may be that you don't want specific sprites to be added to the room in the final game. If that is the case, then simply un-checking the "Export" checkbox to the left of the sprite in the layer properties list will prevent it from being exported as part of the executable. It is important to note, however, that if you have any code that references that asset, then the game will not run correctly, so use this feature with care.
