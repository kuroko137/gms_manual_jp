拡張機能エディタ

拡張機能パッケージを利用すると、GameMaker Language (GML) に関数や定数を追加したり、ゲームのアセットおよびプロジェクトを他の人たちと簡単にやり取りできるようになります。新しい機能を追加する拡張機能であれば、他の言語 （ Android 向けの Java など）または GML で書かれたソースコード、さらには外部 SDK を自分のゲームにリンクさせることも可能です。

サードパーティの拡張機能はおもに GameMaker マーケットプレイスにあります。ここで拡張機能をサブスクライブすると、マーケットプレイスライブラリのウィンドウに登録されてインストール可能となります。また、マーケットプレイスの拡張機能パッケージでない場合も、アセットブラウザを右クリックして「既存のアセットを追加」を選び、場所を指定すればプロジェクトに追加できます。

前バージョンの GameMaker Studio で作成された*.GMEZ ファイルの場合、右クリックメニューから「拡張機能のインポート」を選んで追加できます。

注意: *.GEX の拡張機能は古い形式であり、GameMaker Studio 2 のプロジェクトには対応していないためインポートできません。

アセットブラウザに拡張機能を追加できたら、名前とバージョン番号、出力させるプラットフォームを設定しましょう。チェックを外したプラットフォームでは、実行ファイルの一部として拡張機能が出力されなくなります。また、拡張機能のプロパティは各プラットフォームに合わせる必要があり、以下に基づいて拡張機能のファイル出力先を編集しなくてはなりません。

  • すべて対象: GameMaker Language （ GML）で作られた GML 拡張機能はあらゆるプラットフォームと互換性があります。
  • Windows (通常, UWP, XBoxOne): Windows の DLL ファイル（ダイナミックリンクライブラリ）を利用できるようになります。Windows ゲーム設定で 「x64 Windows ランタイムを使用」にチェックした場合は、必ず 64bit 互換の DLL だけを利用してください。32bit の DLL は 64bit の実行ファイルでは動作しません。
  • macOS: macOS をターゲットにすると標準的な DyLib ファイルを拡張機能に利用できます。
  • Ubuntu (Linux): Ubuntu がターゲットの場合、拡張機能には SO ファイルを利用しなくてはなりません。
  • HTML5: これらのプラットフォームでは JavaScript の拡張機能を利用できます。
  • Android: このプラットフォームでは Java または Android Framework SDK で書かれた拡張機能が必要となります。
  • iOS / tvOS: このプラットフォーム用の拡張機能には Objective C で書かれた*.mm および*.h ファイル、フレームワークを使用している場合、コンパイル済みの*.a ネイティブファイルが必要となります。
  • PlayStation: PlayStation 3 および 4 では PRX ファイルの拡張機能にする必要があり、この PRX ファイルはそれぞれの仕様 （PS3 および PS4）に合わせなければなりません。PSVita 用であれば SUPRX の拡張機能にしてください。

注意: 一部の拡張機能では GameMaker 側でプラットフォーム、専用の CPU アーキテクチャを正しく識別させるために特定の命名規則を守る必要があります。詳しくは拡張機能の作り方の該当項目を参照してください。

1 つの拡張機能にはさまざまな関連ファイルを追加できるため、複数のプラットフォーム用の拡張機能を作ることも可能です。このような場合、メインウィンドウで互換性のあるプラットフォームをすべて選択（横のチェックボックスにチェック）し、拡張機能の各ファイルのプロパティから出力させるプラットフォームを設定しましょう。

拡張機能のファイルまたは Placeholder のターゲットプラットフォームを変更するには、まず拡張機能のプロパティを開き（アセットブラウザの拡張機能をダブルクリックするか、右クリックして「プロパティを開く」を選ぶ）、右側のリソースファイルから編集したいファイルを選んでください。

メモ: macOS または Ubuntu で出力する場合、「zip ファイルとして展開」にチェックを入れると拡張機能のファイル権限を保つことができます。

拡張機能をインストールした後は、GameMaker Language に組み込まれているものと同じように関数・定数を利用できます。クロスプラットフォームのゲーム （Windows と Android の両方で動作させるなどの場合）では不具合を防ぐため、拡張機能の関数を呼び出す前に OS 定数・関数を使ってプラットフォームを確認する必要があります。以下はコードの例です。

switch(os_type)
    {
    case os_android:
        call_android_extension_function(x, y, z);
        break;
    case os_windows:
        call_windows_extension_function(x, y, z);
        break;
    }

なお、異なるプラットフォーム間でも関数名は共有可能です。たとえばメッセージを表示させる拡張機能ファイルが Windows と Android で別々に用意されている場合、それぞれのファイルをプラットフォーム別に出力させるよう設定すれば同じ関数名にできます。

YoYo Games のサードパーティ拡張機能

レガシーバージョンの GameMaker には多くのサードパーティ機能が組み込まれていましたが、サードパーティによって機能が変更されるたびに IDE とランタイムを更新させなくてはならないという非効率的で維持しづらいものでした。そのため現在のバージョンでは拡張機能パッケージの形でサードパーティの機能に対応しており、必要な機能だけを追加したり、目的に合わせて編集するといったことも可能となっています。

このような拡張機能の多くは、特定のゲームオプション （Facebook や Google Mobile Ads など）を選択している場合にのみダウンロード / インストールされます。詳しくは YoYo Games Knowledge Base の各プラットフォーム説明を参照してください。

  • GameMaker Studio 2 の各プラットフォームとそのセットアップ

拡張機能そのものはマーケットプレイスのページにあります。

  • YoYo Games のマーケットプレイスアセット

なお、YoYo Games の拡張機能にはデモとヘルプファイル（*.pdf 形式）が付属していますので、このマニュアルでは紹介しません。
