スプライトエディタ

スプライトエディタでは、スプライトの一般設定を行うことができます。エディタを開くにはスプライトを新規作成（アセットブラウザを右クリックして「スプライト作成」を選択）し、作成されたスプライトをダブルクリックしましょう。または、右クリックメニューから「プロパティ」を選択することでエディタを開けます。

スプライトエディタは、以下の領域に分かれています。

 

スプライト名スプライト名

ここにはスプライトの名前を入力できます。スプライト名は、プロジェクト内でスプライトをリソースとして特定するための名前です。英数字または "_" 記号のみ利用可能ですが、数字で始まる名前にすることはできません。これは GML と DnD™ から参照できない無効なリソース名となってしまうためです。

 

イメージのプロパティイメージのプロパティ

同じ場所にある「スプライトサイズ」ボタンをクリックすると次のウィンドウが開き、イメージ（画像）のサイズを設定することができます。

ウィンドウには変形適用後のプレビューが表示されます。下側の 2 つに分かれた箇所では、次のようにスプライトの新しいプロパティを設定できます。

  • イメージのスケーリング: スプライトの描画キャンバスをリサイズするとともに、それぞれのイメージを指定したサイズに合わせて伸縮させます。イメージのアスペクト比を維持させるか、ピクセル単位とパーセント単位のどちらで拡大縮小させるかを指定可能です。イメージの形式によっては、リニア補間を有効 / 無効にすることもできます（リニア補間とはピクセル全体を「滑らかに」する機能であるため、ドット絵であれば無効にしておくべきでしょう）
  • キャンバスのリサイズ: 描画キャンバスのリサイズのみでイメージは拡大縮小しません。矢印をクリックすると「基準点」を設定してからサイズを変更できます。アスペクト比の維持を切り替えたり、ピクセルかパーセンテージからリサイズ単位を選択できるオプションもあります。

納得できるサイズに設定したら「適用」をクリックしましょう。変更を適用し、サイズ変更プロパティのウィンドウを閉じることができます。なおスケーリングまたはリサイズを適用する前に、適用ボタンの横にある原点へ適用をクリックすると、変更をスプライトの原点に適用するかを選択できます。チェックを入れた場合、原点がカスタム値かつロックされていなければ、新しいサイズに合わせて原点の位置が調整されます。外した場合、原点にスケーリングは適用されません。原点がカスタム値ではなくロック中の場合、オプションはグレーアウトして利用できなくなります。スプライトエディタ設定を開けば、すべてのスプライトの設定値を変更することも可能です。

注: このオプションはビットマップスプライトでのみ利用でき、SWF またはスケルトンアニメーションスプライトではグレーアウトします。

また、この領域ではイメージエディタを開いてスプライトを描いたり、「インポート」ボタンをクリックして作成済みのスプライトをインポートすることができます。一般的なビットマップ形式（ PNG 、JPG 、GIF など）のスプライトをインポート可能で、GameMaker Studio 2 はそれ以外にもベクタースプライト（ SWF ）、スケルタルアニメーションスプライト（ Spine ）に対応しています。これらの形式についての詳細は、非ビットマップスプライトのセクションを参照してください。なお、ビットマップスプライトをインポートする際にファイルエクスプローラーで複数の画像ファイルを選択すると、それぞれスプライトのサブイメージ（フレーム）として別々に追加することができます。

 

テクスチャ設定テクスチャ設定

ここでは GameMaker Studio 2 側でどのようにしてスプライトのイメージを、各デバイス・ブラウザのテクスチャページに保存させるかを設定できます。デスクトップ・プラットフォームではあまり重要なオプションではありませんが、モバイルまたはウェブ向けに開発している場合、イメージアセット（テクスチャ）を適切に管理することはとても重要です。テクスチャの管理が甘いと、パフォーマンスの悪影響に繋がるためです。

水平に並べて表示、垂直に並べて表示のチェックボックスは、タイルスプライトが必要となる場面が少ないためデフォルトではチェックされていません。もっとも、タイル化しておきたい場合はチェックを入れるべきでしょう。ルームまたはビューのスケールを変更させるなら尚更で、テクスチャページが適切に生成されていないのにスケールを変更してしまうと、ゲーム内のグラフィックに乱れが生じてしまいます。

スプライトを 3D ゲームのテクスチャマップとして扱うなら、テクスチャページの分割にチェックを入れ、専用のテクスチャページが与えられるようにする必要があります（各サブイメージにも専用のテクスチャページが与えられ、イメージが 10 ある場合、テクスチャページが 10 生成されます）。このオプションはゲームに必要なテクスチャメモリを増加させるため、十分に注意して扱ってください。また、このオプションを 3D において有効にしている場合、テクスチャは 2 の累乗 （128x128 、256x256 、512x512 など）になります。

次のオプションはイメージの事前乗算アルファです。有効にした場合、スプライトの各サブイメージのアルファが事前乗算されるようになります。通常はサーフェスにスプライトを描画する、または特殊なエフェクトを使用する場合にのみ必要となり、普通の使い方をするスプライトで事前乗算を有効にしたとしても目立った変化は現れません。

もう 1 つのオプションは輪郭フィルタリングです。このオプションはゲーム内でリニア補間が有効にされている際、スプライトの周りに表示される色の「輪」を消すためのオプションです。この問題は、テクスチャページのソーステクセルがハードウェア側で同時に複数補間された場合に発生します。ゲームの解像度を上げるとよりはっきり認識できるでしょう。オプションを有効にしておくと、周辺ピクセルの色がその近くの最大アルファ値ピクセルの色になるようフィルタリングされるため、輪を表示させることなく補間済みピクセルのブレンドが行われるようになります。

最後はスプライトリソースが含まれるテクスチャグループの指定です。テクスチャグループ（テクスチャグループマネージャーであらかじめ定義します）とは、ルームないしレベル用のイメージリソースをまとめて記憶させるために設定するグループです。たとえば、レベル 1 用のイメージを特定のテクスチャグループに含め、レベル 2 用のイメージをそれとは別のテクスチャグループに含めるといったような設定をします。グループに入れられたリソースは同じテクスチャページへの配置が試みられるため、現在のターゲットプラットフォームでゲームが実行されている際に起きる、テクスチャページのスワップ（切り替わり）を減らすことができます。

注: テクスチャグループの指定は必須ではなく、利用することでパフォーマンスを向上できるかはターゲットデバイスが CPU 依存か GPU 依存かによって異なります。

テクスチャページについてはこちらで詳しく説明されています。

テクスチャグループが関わってくるのはビットマップイメージのみで、非ビットマップイメージはテクスチャグループに割り当てられず、上記の各オプションも無効となります。専用のテクスチャページ（スケルトンアニメーションスプライト）をもつ非ビットマップスプライトも存在しますが、それ以外はベクターで構成されたスプライト （SWF スプライト）のためテクスチャを必要としません。詳しくは以下のセクションを参照してください。

コリジョンマスクコリジョンマスク

スプライトには、スプライトを割り当てたインスタンスが衝突を検出するための範囲となるコリジョンマスクがあります。まずはコリジョンマスクのモードを設定しましょう。

  • 自動: スプライトのアルファ値をもとに GameMaker Studio 2 側でコリジョンマスクが自動計算されます（下にある許容値の設定を基準とします）
  • イメージ全体: スプライトの上下左右の境界線と同じ範囲になるようマスクが計算されます。
  • マニュアル: 上下左右の入力値からコリジョンマスクの境界線を設定したい場合に選択します。

コリジョンマスクのモードの他に、ドロップダウンメニューからコリジョンマスクの形を指定することができます。

  • 長方形: シンプルな長方形のコリジョンマスクで、もっとも素早く計算されます。
  • 回転あり長方形: 同じく長方形のコリジョンマスクですが、スプライトを割り当てたインスタンスの image_angle 値によって回転します。通常のコリジョンマスクより計算が遅いものの、精密コリジョンマスクよりは高速です。
  • 楕円形: 楕円形のコリジョンマスクが作られます。長方形マスクより計算が遅いです。
  • ひし形: ひし形のコリジョンマスクが作られます。長方形、楕円形のマスクより計算が遅く、CPU オーバーヘッドも増しています。
  • 精密: スプライトの輪郭線をもとに精密なコリジョンマスクが作られます。サブイメージが複数ある場合、すべてのサブイメージの周辺部分を重ね合わせたマスクが作られます。モードがマニュアルかつ入力値を変更した場合、精密マスクはバウンディンボックスの範囲内に収まるよう切り抜かれます。このモードは非常に低速で、CPU オーバーヘッドが大いに生じます。
  • フレーム毎に精密: 1 フレームごとに新しく精密コリジョンマスクを生成します（それぞれのフレームの輪郭線と同じ形になるよう変化）。モードがマニュアルかつ入力値を変更した場合、精密マスクはバウンディンボックスの範囲内に収まるよう切り抜かれます。もっとも低速なコリジョンマスクのため、非常に例外的なケース以外では使わないようにするべきです。

これらのコリジョン判定オプションはとても重要なものです。オブジェクト同士のインタラクトに直接影響し、設定を間違えればゲーム全体のパフォーマンスに悪影響を与えます。なぜかというと、有効なマスクのある 2 つのインスタンスが接触するたびに、バウンディングボックスとマスクの重なりをチェックする衝突イベントが発生するからです。これはマスクが精密かどうか、イメージのインデックスと適合しているかどうかは関係しません。下の画像はその説明となります。

コリジョンマスク設定の下にあるのはアルファ許容値のスライダーで、ピクセルの透明度を反映させたマスクを生成する際の許容範囲を示しています。許容値を上げると完全に透明でないピクセルはマスクの外側に残り、許容値を下げるとそのようなピクセルもマスクに含まれるようになります。このオプションはバウンディングボックス（マニュアルでない場合のみ）およびマスクの精密コリジョン判定に影響します。

あとはマスクのバウンディングボックスの境界線を設定するだけです。バウンディングボックスとはスプライトのマスクが収まる長方形の領域のことで、このバウンディングボックスの内側にあるピクセルだけがマスクの対象となり、コリジョンの判定に影響します。デフォルトではアルファ許容値を参照しながら自動計算され、さらに「フレーム毎に精密」がチェックされている場合は、サブイメージごとに分けて計算されます。自分で設定する場合は、入力欄に上下左右の値を入力するか、プレビューウィンドウのバウンディングボックスをスプライトの上にドラッグして引っぱってきてください。バウンディングボックスをドラッグすると、端の数値に収まるようマスクが自動生成され、バウンディングボックスの中で切り抜かれます。

覚えておきたい点として、プレビューウィンドウでは四隅にある「ハンドル」をドラッグしてコリジョンマスクを調整できます。ハンドルを操作すると、コリジョンマスクと同時にバウンディングボックスの入力値も変化します。

 

ブロードキャストメッセージブロードキャストメッセージ

スプライトのタイムラインにブロードキャストメッセージを追加できるボタンで、ダイアログを開いて送信するメッセージを指定できます。メッセージはフレームビューの再生ヘッド位置に追加されます。このメッセージは単純な文字列で、Broadcast Message イベントが作動したときに取得されます。

追加したメッセージはフレームビューに表示され、右クリックするとメニューから削除または編集することができます。ブロードキャストメッセージについての詳細はこちらを確認してください。

 

フレーム速度フレーム速度

この領域では、アニメーションフレームが再生される速度を左側の入力欄から設定できます。フレーム速度は、「 1 秒ごとに 1 フレーム」または「 1 ゲームフレームごとに 1 フレーム」という基準から設定します。「フレーム / 秒」で 30 に設定した場合、表示されるのは 1 秒ごとに 30 フレームとなります。「フレーム / ゲームフレーム」で 30 に設定した場合は、ゲーム内の 1 ステップごとに 30 フレーム表示させます。「フレーム / ゲームフレーム」の場合、通常は 1 以下に設定されます（フレーム / ゲームフレームで速度が 0.5 なら、2 ステップごとに新しいフレームが表示されます）

It's important to note that the Frame Speed settings will not only affect the playback of the animation in the preview window, but will also govern how the sprite is animated in your games. The speed you set here will show the sprite preview as it will run in the game with an image speed multiplier of 1, but this can be set in code using the image_speed variable meaning the sprite can run faster or slower depending on what is programmed. So, for example, if you set the sprite to use 10 "frames per second" and then set the image_speed to 0.5, the game will draw the sprite at half speed - 5 frames per second. Conversely, if we set the image_speed for the same sprite to 2, in the game it will be drawn animated twice as fast - 20 frames per second. Note that the frame speed can also be set in the Image Editor.

 

スプライトの原点スプライトの原点

スプライトエディタの上段にあるバーからはスプライトの原点を設定できます。原点とは、ルームに置かれたスプライトの位置となるポイントのことです。たとえば任意の x/y 座標にインスタンスを生成した場合、スプライトの原点がその位置に重なります。

デフォルトでは左上の境界が原点として設定されますが、中央にしたほうが使いやすい場合もあったり、それ以外にも、2D プラットフォームのキャラクターだから下に設定しておきたいということもあるでしょう。原点のドロップダウンメニューをクリックして開けば、こうしたデフォルトの原点を指定することができます。また、スプライトのプレビュー画像をクリックすると、クリックした位置にマーカーが移動して原点をマニュアル設定できます。x と y の入力欄に値を入力することでも設定可能です。なお、数値をマイナス（左と上方向）にしたり、スプライトの縦と横幅よりも大きく（右と下方向）すれば原点をスプライトの領域外に置くことができます。これは複数のスプライトを描画させたいオブジェクトを扱うときに役立つテクニックです。

 

ズーム操作部ズーム操作部

プレビューウィンドウの右上にあるのはズーム操作部です。ボタンを押すと、プレビューのズーム倍率を変更できます。ズームインとズームアウトに加えて、このボタンをクリックすればプレビューを等倍表示にリセットすることができます。また、ウィンドウにフィットボタンをクリックした場合はプレビュー画面をスプライト全体が収まるように合わせます（きちんと収まるようにズームイン / ズームアウトします）。 / を押しながらマウスホイールでもズームイン・ズームアウトでき、 / + でプレビューを等倍に戻してスプライトを原寸表示できます。

 

プレビュープレビュー

スプライトのプレビューウィンドウには、ゲーム内と同じようにスプライトが表示されます。マウスホイールでズームイン・ズームアウトができ、中央クリックでドラッグ、または + を押しながらドラッグで画面の移動ができます。さらにフレーム操作部（以下で説明）からは、プレビュー画像の表示方法を変更可能です。

このウィンドウにはスプライトの原点を示す「十字マーク」が表示され（詳しくは上記のスプライトの原点を参照）、編集中であればコリジョンマスクも表示されます（上記のコリジョンマスクを参照）。

 

再生操作部再生操作部

再生操作部ではプレビューウィンドウを通してフレームのアニメーションを再生できます。再生ボタンでアニメーションの開始 / 終了、ループボタンでスプライトをループ再生させるかどうか（最終フレームになったら 1 フレーム目に戻る）、ピンポン再生させるかどうか（最終フレームになったら逆に進む）を切り替えられます。

Note that the loop/ping-pong option will affect how the sprite is displayed in the game, so setting these options in the sprite will change the way it is shown when the game is compiled and run. However, this is not true for loop-points that can be set in the Frame View (see below for more information) - these are purely visible in the sprite editor and designed to help you test out your animations, but will have no effect on how the sprite animation is displayed in your games.

An additional part of the Frame Controls is the current frame display and input a bit further over on the left. The box shows the total number of frames of the sprite animation as well as the current frame being shown. You can change the current frame at any time by clicking on the input box and typing in the frame number you want to view.

IMPORTANT! Frames are shown numbered from 1 to maxframe number, however in your game code, frames will be numbered from 0 to maxframe number - 1.

FramesFrames

This part of the sprite editor shows each of the different frames that make up a single sprite set along a sequence timeline. You can click on any frame and drag it left or right to change the order in which they will be animated, and you can also double click on any frame to launch the Image Editor and edit them. You can cut, copy and paste frames using the standard keyboard shortcuts or by using the right-mouse button menu options, and you can also select multiple frames to operate on using / + .

The right mouse button menu has the following options:

  • Add Frame - Add a new frame to the animation at the end.
  • Insert Frame - Insert a new frame in the animation after the currently selected frame.
  • Cut / Copy / Paste - Cut, copy and paste one or more frames.
  • Delete Selected Frames - Delete the currently selected frame or frames.
