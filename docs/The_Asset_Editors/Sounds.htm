サウンドエディタ

サウンドエディタでは新しいオーディオリソースにサウンドを読み込み、プロパティを設定することができます。サウンドを読み込むには、サウンド名の右側にあるサウンド読み込みボタンを押してください。ファイル選択ダイアログが開き、サウンドファイルを選択できます。読み込み可能なサウンドファイルは WAV 、MP3 、OGG ファイルとなります。*.wav は容量が大きくなりがちなファイルではあるものの、再生にあたって特定のデコード処理を必要とせず、即座に再生できるという特徴があるため、一般的に短いサウンドエフェクトに使用されます。*.mp3 および*.ogg 形式のファイルは、BGM や、より長く容量が大きいサウンドエフェクトに使用されます。これらのファイルは wav ファイルよりも大幅に少ない容量ですが、再生する前にデコードを必要とするため、わずかに CPU オーバーヘッドが生じます。

ファイルが読み込まれている状態で再生ボタンを押すと、サウンドをプレビュー再生できます。ウィンドウの下側にあるスライダーを移動させると音量を変更できますが、プレビュー再生することで音がどのくらいの大きさかをすぐに確かめることができます。また、再生停止ボタンをクリックすれば好きなタイミングでプレビューを停止可能です。

サウンドウィンドウには、他にも以下のようなオプションがあります。

 

名前とファイル名前とファイル

ここにある開くボタンをクリックすると、ファイルエクスプローラーから音声ファイルを読み込むことができます。左部分の「名前欄」からはリソースの名前を設定可能です。名前には英数字と "_" 記号のみ使用可能で、数字から始まる名前をつけることはできません。

 

属性属性

効果音（ WAV 形式）であれば素早く再生するため、デコードを必要としない「非圧縮」にするべきですが、OGG または MP3 の場合、3 つのオプションのうちいずれかを選ぶといいでしょう。

オーディオを圧縮すると、あらゆるサウンドファイル（*.wav 、*.mp3 、*.ogg のいずれも区別せず）をすべてのプラットフォームにおいて強制的に、Ogg Vorbis （*.ogg ）形式のファイルに圧縮します。圧縮されたサウンドは容量が小さくなるものの、再生する前に展開してメモリに読み込まなくてはならないため、わずかに CPU オーバーヘッドが生じます。そのため CPU 負荷の高いゲームで圧縮を利用する場合、その点を踏まえておいてください。

読み込み時に展開を指定すると CPU オーバーヘッドを軽減できます。この方法では、サウンドをデバイスのメモリに置くため高速で再生できるようになりますが、そのぶんメモリの消費量は増加します。

オーディオの圧縮を選んだ場合、さらにサウンドをディスク上からストリーミング再生するかを指定しなくてはなりません。ストリーミング再生されるサウンドは展開されずにリアルタイムで再生され、メモリに読み込むことなくディスク上からストリーミングされます。ストリーミングはファイル展開時に一度だけ生じる、ゲームを一時停止させるようなオーバーヘッドを抑えられるため BGM に適していますが、CPU 負荷がより少ないシンプルな効果音にはお勧めしません。オーディオのストリーミングを選ぶとサウンドをオーディオグループに割り当てることができなくなり、オーディオグループの設定は無効となります。ストリーミングサウンドは他のサウンドと同じようにパッケージングされませんが、コードを通して読み込み、再生、読み込み破棄のコントロールが可能です。

It is important to note that when targeting the HTML5 platform. streamed audio will not play on iOS when the play request does not directly originate from a user interaction (i.e: a touch event, etc.). However, since GameMaker Studio 2 queues up click events and only handles them on the next frame, it is impossible to play streamed audio without the browser blocking it. To get around this, any HTML5 game run on an iOS browser will treat all streamed audio as unstreamed in the engine, with the exception of streamed audio not being preloaded.

What this means is that if your game is running on an iOS browser, and - for example - you have 10 music tracks that are all set as streamed mp3 files. All 10 will be initialised like unstreamed audio would, but they won't be decoded immediately before the game loads, unlike normal streamed audio which would get preloaded/decoded immediately during the load screen phase.

While this bypasses the iOS issue and prevents any huge increase in initial load times, it does mean that it may result in a state where your game has loaded, but certain streamed sounds may still be downloading or decoding. To deal with this you can use the GML function audio_sound_is_playable(), which checks to see if a given sound can actually be played. On HTML5 this will return false if the sound is not fully loaded or decoded yet, and true if it is and can be played (on all other platforms it will always return true).

 

VolumeVolume

Here you can set the playback volume for the given sound. This is value is simply the default value where if the slider is in the middle - for example - then volume for that sound when used in the game will be 0.5. However, this will be overridden by any change to the gain using the audio functions in your project. Beside the volume controls you also have the Playback controls where you can preview the sound, set it to loop or rewind. You can also click the Icon to open the sound in the Sound Mixer.

 

Target OptionsTarget Options

The Target Options panel is where you set the conversion rates for your game. These settings will affect how the sound is exported for the target platform, and GameMaker Studio 2 will automatically convert your sound files for you - you simply control the final quality of those sounds when converted. Generally the default settings of 16 bit quality (or bit depth), 44100kHz sample rate and 192kbps bit rate are perfectly adequate, but it may be that you wish sound effects (for example) to have a lower quality and music to have a higher quality, in which case these settings can be changed.

You can also choose whether the conversion should be mono, stereo or 3D. If you plan on using the audio emitter functions to create 3D sound effects, you should indicate this here, otherwise choose mono or stereo depending on the use that the sounds are going to be put to. If you choose to use the sound for 3D audio, it will be exported as a mono sound.

NOTE: The higher the quality of the sound, the larger the file, and this should be taken into account when deciding what bitrate and sample rate to choose.

 

Audio GroupAudio Group

You can assign your audio file to a specific Audio Group. These are created from the Audio Groups window and are used to group audio assets together for easier memory management and volume control.

Once an audio file has been assigned to a group you can then use the specific audio group functions to change them and lo load/unload them from memory. See the reference section on Audio Functions for further information.

If you have selected streamed audio then this option will be disabled, as streamed sounds are not packaged in the same way as other sounds and will be not be loaded into memory, but instead streamed directly from the storage medium.

Note that while the rest of the audio properties are configuration specific, the audio group setting is not and a sound will belong to the same group on all configurations that you use.
