サウンドエディタ

サウンドエディタでは、作成したオーディオリソースにサウンドを読み込んだり、プロパティの設定を行うことが可能です。サウンドを読み込むには、名前の右側にあるサウンド読み込みボタンを押しましょう。ファイル選択ダイアログが開き、WAV 、MP3 、OGG 形式のファイルを選ぶことができます。*.wav は容量が大きくなりがちなファイルですが、再生にあたって特定のデコード処理を必要とせず、即座に再生できるという特徴があるため、おもに短い効果音に使われます。*.mp3 および*.ogg 形式のファイルは BGM や、長めの容量が大きい効果音に使われるファイルです。これらのファイルは wav ファイルよりも容量がずっと少ないものの、再生する前にデコードを必要とするため、わずかな CPU オーバーヘッドを生じさせます。

ファイルを読み込ませてから再生ボタンを押すと、サウンドをプレビュー再生できます。ウィンドウの下側にあるスライダーを動かすとサウンドの音量を変更できますが、プレビュー再生しながら動かせば音の大きさをそのまま確かめることができます。再生停止ボタンをクリックすると、好きなタイミングでプレビューを停止できます。

サウンドウィンドウには、他にも次のようなオプションがあります。

 

名前とファイル名前とファイル

開くボタンをクリックすると、ファイルエクスプローラーから音声ファイルを読み込むことができます。左の「名前欄」ではリソースの名前を設定できます。名前には英数字と "_" 記号のみ使用可能で、数字から始まる名前をつけることはできません。

 

属性属性

ここではサウンドの属性を指定できます。属性はゲームがエクスポートされたときに設定され、実行時のサウンド再生方法を変化させます。効果音（ WAV 形式）ならすぐに再生されるようにデコードを必要としない「非圧縮」にするべきですが、OGG または MP3 の場合は、他の 3 つの設定のうちいずれかを選ぶ必要があるでしょう。

オーディオの圧縮は、あらゆるサウンドファイルを（*.wav 、*.mp3 、*.ogg の区別をせず）すべてのプラットフォームにおいて強制的に、Ogg Vorbis （*.ogg ）形式のファイルに圧縮します。圧縮されたサウンドは容量が小さくなるものの、再生する前に展開してメモリに読み込まなくてはならないため、わずかな CPU オーバーヘッドを生じさせます。CPU 負荷の高いゲームで圧縮させる場合、その点を踏まえておいてください。

「読み込み時に展開」を指定すると CPU オーバーヘッドが軽減され、デバイスのメモリ上にサウンドを置くことで素早く再生できるようになりますが、そのぶんメモリの消費量は増加します。

オーディオを圧縮する場合、サウンドをディスク上からストリーミング再生するかを決めなくてはなりません。ストリーミング再生ではサウンドを展開してメモリに読み込むのではなく、ディスク上からリアルタイム再生します。ファイル展開時に一度だけ生じる、ゲームを一時停止させるような負荷を抑えられることから BGM に向いており、CPU 負荷がより少ないシンプルな効果音では非推奨となります。また、ストリーミングを選ぶとサウンドをオーディオグループに割り当てることができなくなり、オーディオグループの設定は無効となります。これは他のサウンドと同じようにパッケージングされなくなるためですが、コードを利用することで読み込み・再生・読み込み破棄のタイミングを自由にコントロールできます。

HTML5 プラットフォームをターゲットにしている場合、注意事項があります。iOS ではユーザーのインタラクション（タッチイベントなど）によって再生リクエストが直接送られない限り、ストリーミング再生が行われません。しかし GameMaker Studio 2 では、キューに入れられたクリックイベントを次のフレームまでしか処理できないため、ブラウザからブロックされずにストリーミング再生を行うことは不可能です。この問題を避けるために、iOS ブラウザで HTML5 ゲームが実行された場合、これらのオーディオはエンジン側では非ストリーミングオーディオとして扱われます（プリロードされた場合を除く）。

具体的に言うと、たとえば iOS ブラウザでゲームを実行したとき、ストリーミング再生される mp3 ファイルの BGM トラックが 10 あったとします。いずれのトラックも非ストリーミングオーディオのように初期化されますが、ゲームが読み込まれる直前にデコードされることはなく、通常のストリーミングオーディオと異なりロードスクリーンに入った瞬間にプリロード / デコードされることになります。

こうすることで iOS の問題を回避でき、初期ロード時間が大幅に伸びることもなくなりますが、ゲームの読み込みが終わっているのにダウンロード中・デコード中となってしまい、サウンドをストリーミング再生できない場合もあります。この問題は audio_sound_is_playable()という GML 関数を利用し、サウンドの再生状態をチェックすることで解決できます。HTML5 でこの関数を使うと、サウンドの読み込みまたはデコードが完全に終わっていない場合は false が返され、再生可能になったら true が返されます（その他のプラットフォームでは常に true が返されます）。

 

音量音量

ここではサウンドの再生音量を設定できます。設定した値は音量のデフォルト値となり、たとえばスライダーをちょうど真ん中に動かした場合、ゲーム内でサウンドを再生したときの音量は 0.5 となります。ただしオーディオ関数によってゲインが変更された場合、そちらの値が優先されます。音量コントロールの横にある再生操作部からはプレビュー再生ができ、プレビューのループ切り替えや再生位置を最初に戻すことができます。また、アイコンをクリックするとサウンドミキサーが開きます。

 

ターゲット設定ターゲット設定

ターゲット設定パネルでは変換レートの設定が可能で、ターゲットプラットフォームにおけるサウンドのエクスポート方式を変化させられます。サウンドは GameMaker Studio 2 側で自動的に変換されるため、設定する必要があるのは最終的な品質だけです。ほとんどの場合ではデフォルト設定の 16bit 品質（もといビット深度）、サンプリングレート 44100kHz 、ビットレート 192kbps で十分ですが、たとえば効果音は低品質に、BGM は高品質にしておきたい場合などは変更するといいでしょう。

他にはモノラル、ステレオ、3D のどちらに変換するかを指定できます。オーディオエミッタ関数から 3D 効果音を生成するのであれば 3D を選ぶべきですが、そうでなければ用途に応じてモノラルかステレオのいずれかを選びましょう。なお 3D オーディオとして設定した場合、モノラルサウンドとしてエクスポートされます。

注: サウンドの品質を上げるほどファイルの容量は大きくなります。よく考えてビットレートとサンプリングレートを選びましょう。

 

オーディオグループオーディオグループ

オーディオファイルには、オーディオグループの割り当てが可能です。このグループはオーディオグループウィンドウから作成でき、オーディオアセットをグループに割り当てることで、メモリの管理や音量のコントロールが簡単にできるようになります。

グループに割り当てられたオーディオファイルは、オーディオグループ関数によって変更を加えたり、メモリへのロード / アンロードが可能となります。詳しくはオーディオ関数のリファレンスセクションを参照してください。

オーディオをストリーミングに指定している場合、このオプションは無効となります。これはストリーミングサウンドが他のサウンドと同じようにパッケージングされず、メモリに読み込まれずにストレージ媒体からストリーミング再生されるためです。

他のオーディオプロパティは構成固有のものですが、オーディオグループは異なり、すべての構成において同一のグループとなります。
