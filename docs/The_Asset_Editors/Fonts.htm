フォントエディタ

ゲーム内でテキストを描画しようとすると標準の Arial 12 ピクセルフォントで描画されますが、より目を引くような、個性あるビジュアルのテキストにするには、別のフォントを使用する必要があるでしょう。コンピューターにある別のフォントを使用するには、GameMaker Studio 2 からフォントアセットを作成しなくてはなりません。フォントアセットは、アセットブラウザを右クリックして「フォント作成」を選ぶと作成できます。フォントアセットを作成したら、コンピューターにある特定の種類のフォントを指定しましょう。アクションまたはコードから、そのフォントを描画用のフォントに設定するとゲーム内に反映できます。

フォントアセットを作成すると、まずフォントエディタのウィンドウが開き、次の項目が表示されます。

詳細詳細

フォントに固有の名前をつけ（英数字および "_" 記号のみ利用可）、使用するフォントを選択します。フォントはコンピューターにインストールされているものでなければならず、さらに GameMaker Studio 2 ではライセンス上の問題から、デフォルトではプロジェクトファイルにフォントファイルそのものが保存されず、一連のビットマップグリフとしてフォントのテクスチャページをレンダリングします。このテクスチャページはテクスチャグループに割り当てられ、グループの選択項目から指定できます。

マーケットプレイスアセットを制作するなど、何らかの理由でフォントファイルそのもの（*.ttf のみ可）をプロジェクトに含めて配布したいなら、プロジェクトにコピーの項目をチェックしてください。エクスプローラーが開きますので、フォントファイルを指定しましょう。そうするとプロジェクトにバンドルされるファイルとして追加できます。

注意！ 必ず自分のプロジェクトで使用または再配布することが法的に許可され、ライセンス上の問題がないフォントのみを利用するようにしてください。

 

プリセットプリセット

この領域ではサイズ（ポイントで指定、デフォルトは 12pt） の設定ができ、フォントの太字、斜体、アンチエイリアスを適用するかを設定できます。右側のプレビューウィンドウには指定したサイズと変形でゲーム内の表示とほぼ近いテキストが表示されます。


範囲管理範囲管理

フォントリソースの作成時、そのフォントのグリフをすべて含める必要はほとんどありません。そのため GameMaker Studio 2 では選択されたフォントのうち 32 から 127 文字目を使用し、これだけで数字、小文字、大文字、特定の句読点をカバーできます。とはいえ、ゲームを最適化するために使用する文字範囲を変更することも可能で、たとえば特定のフォントの数字だけを使いたいなら、数字だけを選択できます。そうするには、まず削除ボタンをクリックして現在の範囲を消去し（プレビューウィンドウをクリックして範囲を選ばなければ削除できない場合があります）、追加ボタンをクリックして新しい範囲を追加してください。すると次のウィンドウが隣に開きます。

このウィンドウには、次のようにフォントを標準範囲に設定できるボタンがあります。

   - 通常範囲: 32 から 127

- ASCII 範囲: 0 から 255

- 数値範囲: 数値 10 文字のみ

- 文字列 大文字および小文字の文字列のみ

異なる範囲にするには、文字列範囲ボックスに最初と最後の文字インデックス範囲を入力してください（範囲中にない文字はスペースに置き換えられます）ボタンと入力ボックスとは別に、次のようにもう 2 つの便利な選択項目があります。

  • コードから検索: ボタンをクリックして OK を押すと、GameMaker Studio 2 からゲームコードの文字列を自動解析し、ゲームの全テキストをカバーできる文字範囲を新しく作成します。なお、すべての文字列（ "" または''の内容）を調べるため、文字範囲にファイルの名前が混ざってしまう場合があります。
  • ファイルから検索: 上にある「コードから検索」と同じような機能をする項目です。クリックすると参照するファイルが求められ、OK を押すと実行されます。GameMaker Studio 2 によってファイルが解析され、見つかったテキストから文字範囲が作成されます。

なお、テキストの範囲はプレビューウィンドウに直接テキストグリフを追加・削除することで手動編集できます。

 

Texture ControlsTexture Controls

As mentioned above (in the section on Font Details) by default all font glyphs will be rendered to a texture page as part of a texture group to convert them into graphical resources, and when the game is compiled the font will be added to a temporary texture to create a single image with all the required font glyphs and then this will be added to the main texture group for exporting. The buttons here are for tweaking how the font texture is generated or for regenerating a new font texture.

The Options button will open up the advanced font texture options:

These options can, sometimes radically, alter how a font will be displayed in the game, and not all fonts will require the same settings. The following options are designed to give you the greatest customisation possible for your font:

  • Hinting: Font hinting (also known as instructing) is the use of mathematical instructions to adjust the display of an outline font so that it lines up with a rasterized grid. At low screen resolutions, hinting is critical for producing clear, legible text, and here you can select from different hinting algorithms to get the best result possible. Default is "normal".
  • Glyph Options: These options can be used to change how the glyphs are loaded by enabling/disabling different operations. These are:
    1. Disable Hinting: This will disable the fonts native hinting and the auto-hinter.
    2. Disable Auto-Hinter: This will disable the auto-hinter and instead use the fonts native hinter (if it has one). Note that this overrides the "Prefer Auto-hinter" option.
    3. Prefer Auto-Hinter: This will tell GameMaker Studio 2 to prefer using the auto-hinter over any native hinting that the font may have. Note that this is overriden by the "Disable Auto-hinter" and "Disable Hinting" options.
    4. Warping: Enabling this will slightly warp glyphs to improve grid alignment. Note that this requires the auto-hinter and a "Normal" hinting mode.
    5. No Scale: When this option is checked it will keep glyphs in font units and ignore the requested size. This implies hinting is disabled.
    The default state for all the above options is disabled.
  • Interpreter: Here you can select the TrueType version that GameMaker Studio 2 should use for font rendering. Default is TrueType v40.
  • Rounding: This option is for setting the rounding method when converting from a fixed point decimal size to an integer. Default is "floor".

The Regenerate button can be used to regenerate the font texture. This is required if you want to update any changes you've made to an existing font, or if you have created a new font and you want to see how it looks in the Preview Window. Note that you can also regenerate fonts from the right button menu for the font (or fonts) in the Asset Browser.

It is important to note that when you first create a new font, if you don't regenerate the texture in the editor, then the texture will be created the first time you run or test your game, but that is the only time that it will be automatically generated. This means that once a font has a generated texture, any changes made to the font resource will require that the font is regenerated for the changes to be visible in the game, otherwise the "old" font texture will still be used.

GameMaker Studio 2 will render the generated font texture as a PNG file along with the rest of the project files, and should you load the project and the base font is not found, then the game will still display text correctly using the stored texture. A missing font will mean, however, that you can no longer change any of its properties, other than its name, font family and texture group (from the Texture Group Editor).


Preview ModePreview Mode

In this section you can choose to have the Preview Window show either some sample text of the font selected, or the rendered texture image that has been generated for the font.

 

PreviewPreview

The Preview Window is where you can see a preview of the font, either as a sample of text, or as a texture. When the Preview Mode is set to Sample, you can type in any text that you want to test directly into the preview window to see how it looks, while selecting Game Preview will show the rendered texture image that was generated for the font. If no image is present, then you should click the Regenerate button to generate the texture.

Note that the texture image can be different to the sample, since the texture will not be regenerated unless explicitly told to. This means that after you have initially generated a font texture, changes made to the font resource will not be carried over to the game unless the texture is regenerated.
