Push Notifications

Push Notification イベントはデバイス OS におけるプッシュ通知のコールバックによって作動し、push_local_notification()関数を通したローカルソース、または自分のサーバーなどのリモートソースを経由します。作動時にはイベント内だけで使える DS マップが生成され、async_load という専用の変数に保存されます（イベントの詳しい使い方については、Asynchronous イベントを作動させる各関数のページにあるサンプルコードを参照してください）。この DS マップには以下のキーが含まれます。

  • "type": 値が "local" であればデバイスへのローカル通知、"remote" であればリモート通知、"register" であればリモート通知の登録となります。
  • "status": 値が "1" であれば成功に、"0" であればエラーになります。

"type" と "status" の戻り値によってキーが追加される場合もあります。"status" の場合、エラー（ "0" ）が返されると以下のキーが追加されます。

  • "error": 受信したエラーの詳細が含まれています。

"status" の値が 1 （エラーなし）の場合、"type" キーの値に応じて DS マップには以下の値が含まれます。

  • "reg_id": "type" が "register" を受信した場合、このキーにはリモート通知をするデバイスの登録 ID が保持されます。
  • "data": "type" が "local" または "remote" を受信した場合、このキーには通知関数を呼び出すときに設定した文字列ペイロードが保持されます。

分かりやすくなるよう、ちょっとしたスニペット例を用意しました。この例では、次のコードによってローカルプッシュ通知を送信しています。

var fireTime = date_inc_day(date_current_datetime(), 1);
var data = "daily_reward";
push_local_notification(fireTime, "Ahoy!", "Catch The Haggis Has A Present", data);

ここではタイマーを設定し、1 日後にデバイスへ通知をプッシュさせます。1 日経つと、ゲームがバックグラウンドまたは起動していない場合であっても指定したタイトル、メッセージの通知がユーザー側で表示され （iOS では名前のみでタイトルは無視されます）、それから Asynchronous Push Notification イベントが呼び出されます。通知が送られたときにゲームがフォアグラウンドだった場合、通知は表示されないものの Asynchronous イベントは作動するためご注意ください。イベントからは以下のようにコールバックを処理します。

var type = ds_map_find_value(async_load, "type");
var status = ds_map_find_value(async_load, "status");
if status == 0
   {
   //error of some kind
   var error = ds_map_find_value(async_load, "error");
 show_debug_message("error=" + string(error));
   }
else
   {
   if type == "register"
      {
      global.reg_id = ds_map_find_value(async_load, "reg_id");
      }
   else
    {
      var data = ds_map_find_value(async_load, "data");
         if data == "daily_reward"
         {
         global.Gold += 1000;
         }
      }
   }
