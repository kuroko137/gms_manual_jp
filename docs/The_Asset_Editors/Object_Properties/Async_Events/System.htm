System

このイベントはシステムレベルイベント（ゲームパッドの検出や、XBox Live への自動サインインなど）からのコールバックによってのみ作動し、async_load 変数に格納された DS マップを返します。この DS マップにはコールバックを作動させたシステムレベルイベントに応じたキー / 値ペアが含まれています。以下はイベントの一覧です。

 

ゲームパッド

ゲームパッドの接続または接続切断によって作動するイベントであり、async_load マップには以下のキー / 値ペアのうちのいずれかが返されます。

  • "event_type" - System イベントの受信タイプであり、以下の文字列のうちのいずれかとなります。
    • "gamepad discovered" - 新しいゲームパッドが接続されたとシステムによって報告された場合
    • "gamepad lost" - ゲームパッドの接続が解除されたとシステムによって報告された場合
  • "pad_index" - 追加または削除されたパッドのインデックス

このイベントがあれば、ゲームパッドの判定ロジックを Step または Alarm イベントから System イベントに移し、実際に必要となるタイミングで実行できるようになります。

重要！このイベントはコードに最低でも 1 つの gamepad_*関数または DnD™ アクションが含まれていないと作動しません。ランナーからゲームパッドのサブシステムが初期化されるのはプロジェクト内で関数が呼び出された場合のみとなるため、ゲームパッドが存在しない場合、ゲームパッドを追加 / 削除しても System イベントは作動しません。

 

仮想キーボード

仮想キーボードが開かれた、または閉じられると作動するイベントであり、async_load マップには以下のキー / 値ペアが返されます。

  • "event_type" - System イベントの受信タイプであり、仮想キーボードの "virtual keyboard status" になります。
  • "screen_height" - 仮想キーボードの高さです（ピクセル単位）。0 にした場合、キーボードは非表示となります。
  • "keyboard_status" - キーボードの現在の状態であり、以下の文字列のうちいずれかが返されます。
    • "hiding"
    • "hidden"
    • "showing"
    • "visible"

仮想キーボードについての詳細はこちらを参照してください。

 

XBox Live

XBox One をターゲットとして UWP 出力を使用し、UWP ゲームオプションの 「XBox Live を有効」オプションにチェックを入れていると作動する非同期システムイベントです。Xbox Live が有効となっている GameMaker Studio 2 の UWP プロジェクトを開始すると、Xbox Live にサイレントモードで自動サインインします。サインインの結果は async_load マップに返され、以下のキー / 値ペアのいずれかとなります。

  • "event_type" - System イベントの受信タイプであり、以下の文字列のうちのいずれかとなります。
  • "user signed in" - サイレントモードでのサインインに成功
  • "user sign in failed" - サイレントモードでのサインインに失敗（このような場合、xboxlive_show_account_picker()関数でユーザーがサインインするアカウントを指定しましょう）
  • "user signed out" - ユーザーがサインアウトした

他の XBox Live 関数からもイベントが作動することがありますが、そうした関数のコールバックについては関連する関数のページで詳しく説明されています。XBox Live で利用できる関数についての詳細は、こちらを参照してください。

 

オーディオシステム状態

このイベントタイプは HTML5 ターゲットでの動作時専用であり、Web オーディオコンテキストの状態が変化するたびに作動します。たとえばループする BGM がある場合、このイベントの作動に応じて一時停止、停止、再開できるということです。async_load マップから以下のキー / 値ペアを調べることで実現できます。

  • "event_type" - システムイベントの受信タイプです。オーディオシステムが初期化されている、またはコンテキストが変更されていると "audio_system_status" という文字列になります。

このイベントタイプで作動した場合、async_load マップには以下のキーが追加されます。

  • "status" - オーディオシステムの状態であり、以下の 2 つの文字列のいずれかとなります。
  • "available" - オーディオシステムが初期化され、サウンドを再生できる状態
  • "unavailable" - オーディオシステムが初期化されていないか、コンテキストが実行中でないため、サウンドを再生できない状態（サウンドを再生するすべての関数で-1 を返されます）

オーディオに利用できる関数については、こちらで詳しく説明されています。

注意: このイベントは HTML5 ターゲット用に設計されています。異なるプラットフォームでも作動はしますが、オーディオエンジンが最初に初期化されるゲーム開始時点でのみ作動します。
