HTTP

HTTP イベントは、http_post_string()といった http_*()関数からのコールバックによって作動するイベントです。作動時にはイベント内だけで使える DS マップが生成され、async_load という専用変数に格納されます（イベントの詳しい使い方については、各関数のページにあるサンプルコードを参照してください）。この DS マップは次のような構造となっています。

  • "id": コマンドから返される ID です。"http_"リクエストを連続して送信した場合、どのリクエストに応答するかを確認する必要があります。最初にリクエストを送信したときに保存された値とこの値を比較すれば、該当するリクエストを調べることができます。
  • "status": 返された値が 0 より小さければエラー、0 なら成功、1 ならコンテンツのダウンロード中となります。
  • "result": 受信したデータ（文字列のみ）です。http_get_file()を利用した場合、ダウンロードされたファイルのパスになります。
  • "url": リクエストされた完全 URL です。
  • "http_status": HTTP ステータスコードそのものです（有効な場合のみ）。大半のブラウザでは、304 Not Modified （未更新）、204 No Content （内容なし）といった、標準の Web ステータスコードが返されます。

上記は http_post_string()関数から取得できる内容となりますが、ほかの http_関数で返される DS マップは少し異なる内容になることがあります。マニュアルからそれぞれの関数の記事を参照し、正確に返されるデータを調べるといいでしょう。

メモ: async_load には DS マップが生成されるため、これらの関数は json_encode()、json_decode()関数と組み合わせると非常に便利です。

ダウンロードするファイルを要求した場合、DS マップにデータが追加されることがあります。その場合は "status" の値が 1 となり、以下のキーが DS マップに含められます。

  • "contentLength": Web サーバーから受信することになるファイルのサイズです（サーバーからデータが返されなかった場合は-1 になります）。
  • "sizeDownloaded": ダウンロード済みデータのサイズです。

このイベントはデータパケットを受信するたびに作動するのではなく、メインゲームループにおけるダウンロード中に任意のタイミングで更新を行います。また現時点でこの機能は、通常の Windows ターゲットプラットフォームでのみ利用可能となるためご注意ください。
