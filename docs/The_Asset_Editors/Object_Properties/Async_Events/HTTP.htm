HTTP

HTTP イベントは、http_post_string()のような http_*()系関数からのコールバックによって作動するイベントです。作動時にはイベント内だけで使える DS マップが生成され、async_load という専用の変数に含まれます（イベントの詳しい使い方については、各関数のページにあるサンプルコードを参照してください）。この DS マップは次のような構造となっています。

  • "id": コマンドから返される ID です。"http_"リクエストを連続して送信した場合、どのリクエストに応答するかを確認する必要があります。最初にリクエストを送信したときに保存された値とこの値を比較すれば、該当するリクエストを調べることができます。
  • "status": 返された値が 0 より小さければエラー、0 なら成功、1 ならコンテンツのダウンロード中となります。
  • "result": 受信したデータ（文字列のみ）、または http_get_file()からの場合はダウンロードされたファイルのパスとなります。
  • "url": リクエストした正式な URL です。
  • "http_status": HTTP の元ステータスコードです（利用可能な場合）。大半のブラウザでは、304 Not Modified （未更新）、204 No Content （内容なし）といった、標準的な Web ステータスコードが返されます。

上記は http_post_string()関数で取得できる内容ですが、http_系列の各関数では少し異なるマップが返される場合があります。そのためマニュアルから各関数の記事を確認し、返される正確なデータを調べるようにしてください。

メモ: async_load では DS マップが作られるため、http_系列の関数は json_encode()および json_decode()関数と組み合わせると特に便利です。

また、ダウンロードするファイルを要求した場合は DS マップにデータが追加されることがあります。その場合は "status" の値が 1 となり、以下のキーが新たに保持されます。

  • "contentLength": Web サーバーから受信することになるファイルのサイズです（サーバーからデータが返されなかった場合は-1 になります）。
  • "sizeDownloaded": ダウンロード済みデータのサイズです。

このイベントは 1 つのデータパケットを受信するたびに作動するのではなく、メインゲームループにおけるダウンロード中に任意のタイミングで更新が行われます。また、現時点でこの機能を利用できるのは通常の Windows ターゲットプラットフォームのみとなるためご注意ください。
