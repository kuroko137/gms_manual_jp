Networking

ネットワークトラフィックを受信するたびに作動するイベントであり、ネットワーク関数と関連しています。作動するとイベント内だけで使える DS マップが生成され、async_load という専用変数に格納されます。この DS マップには、Networking イベントのタイプに応じたさまざまな情報が含まれます。

 

Networking イベント共通

これらのキーは受け付けるすべてのネットワーク関数で共通となっており、async_load の DS マップに必ず追加されます。

  • "type": 以下のテーブルにある定数のいずれかが戻り値となり、Network イベントのタイプを表します。
  • "id": イベントが受信したソケット ID で、network_create_server() または network_create_socket() 関数によって返される実数です。ソケット ID として返される値はほとんどの場合、イベントを作動させた TCP または UDP ソケットの ID となりますが、イベントがサーバー側で作動し、なおかつ Data イベント（以下を参照）である場合、データを送信したクライアントのソケット ID となります。
  • "ip": ソケットの IP アドレス（文字列）です。
  • "port": IP アドレスに割り当てられているポートです。UDP での動作時に役立ちます。

"type" キーの戻り値は、以下の 4 つの定数から選ばれます。

定数 説明
network_type_connect イベントが接続によって作動
network_type_disconnect イベントが切断によって作動
network_type_data イベントがデータの受信によって作動
network_type_non_blocking_connect イベントが非ブロック設定の接続によって作動（ network_set_config()から設定が、network_connect_async()または network_connect_async_raw()から接続ができます）

 

接続 / 切断

タイプが network_type_connect または network_type_disconnect のイベントがある場合、async_load の DS マップには次のキーが含まれます。

  • "socket": 接続 / 切断されたソケット ID を保持するキーです。

イベントのタイプが network_type_non_blocking_connect の場合、さらに次のキーが含まれます。

  • "succeeded": 0 か 1 になり、0 の場合は接続のタイムアウト、1 の場合は接続に成功し、ソケットが利用可能であることを意味します。

Networking イベントは、サーバー切断時にはクライアント側では作動しないためご注意ください。また、network_type_*のイベントであっても接続中のサーバーが切断された場合は作動しません （TCP による接続時も同様です）。

 

データの受信

イベントのタイプが network_type_data の場合、ネットワークがデータを受信したことになり、マップには以下のキーが含まれます。

  • "buffer": イベントによって生成される固有の「バッファ ID」です。増幅 （grow）タイプの 1 バイトアライメント式バッファが変数に格納されるバッファ ID を保つために作られ、同イベントにおける以降のバッファに対する関数呼び出しで利用されます。async_load マップと同じように、生成されたバッファはイベントの終了後、メモリから自動で削除されます。バッファについての詳しい情報は、リファレンス - バッファを参照してください。
  • "size": 受信中のバッファデータのサイズです（バイト単位）。

注意: 生成されたバッファはイベント内でのみ有効となり、イベントが終了すると解放されます。
