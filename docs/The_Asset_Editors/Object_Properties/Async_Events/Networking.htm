Networking

Networking イベントはネットワークトラフィックを受信すると必ず作動するイベントであり、ネットワーク関数と関連しています。作動するとイベント内だけで使える DS マップが生成され、async_load という専用の変数に保存されます。この DS マップには、Networking イベントのタイプに応じたさまざまな情報が含まれます。

 

共通 Networking イベント

以下のキーは受信したすべてのネットワーク関数で共通となっており、async_load のマップに必ず追加されます。

  • "type": 以下のテーブルに並んでいる定数のいずれかが戻り値となり、Network イベントのタイプを示しています。
  • "id": イベントが受信したソケット ID （ network_create_server または network_create_socket 関数によって返される実数）です。返されるソケット ID は多くの場合、イベントを作動させた TCP または UDP ソケットの ID となりますが、イベントがサーバー側で作動し、なおかつ Data イベント（以下を参照）である場合、データを送信したクライアントのソケット ID となります。
  • "ip": ソケットの IP アドレス（文字列）です。
  • "port": IP アドレスに割り当てられているポートです （UDP での動作時に有用）。

"type" キーの戻り値は 4 つの定数から選ばれ、以下はその一覧です。

定数 説明
network_type_connect イベントが接続によって作動
network_type_disconnect イベントが切断によって作動
network_type_data イベントがデータの受信によって作動
network_type_non_blocking_connect イベントが非ブロック設定の接続によって作動（ network_set_config()から設定でき、network_connect_async()または network_connect_async_raw()によって接続できます）

 

接続 / 切断

network_type_connect または network_type_disconnect タイプのイベントの場合、async_load のマップには以下のキーが追加されます。

  • "socket": 接続 / 切断されたソケット ID を保持するキーです。

イベントのタイプが network_type_non_blocking_connect の場合、さらに以下のキーが追加されます。

  • "succeeded": 0 か 1 になるキーであり、0 の場合は接続のタイムアウト、1 の場合は接続に成功し、ソケットが利用可能であることを意味します。

Networking イベントはサーバーが切断されてもクライアント側では作動せず、network_type_*イベントにおいても接続中のサーバーが切断され、TCP 基準の接続であってもクライアント側では作動しないためご注意ください。

 

データの受信

イベントのタイプが network_type_data の場合、ネットワークがデータを受信したことになり、マップには以下のキーが含まれます。

  • "buffer": イベントによって生成された固有の「バッファ ID」です。"grow" タイプの 1 バイト単位で揃えられたバッファが生成され、変数にはその ID が保持されます。このイベント内の別の関数からバッファを呼び出すときに便利です。async_load マップと同じように、生成されたバッファはイベントの終了後、メモリから自動で削除されます。バッファについての詳しい情報は、リファレンス - バッファを参照してください。
  • "size": 受信したバッファデータのサイズです（バイト単位）。

注意: 生成されたバッファはこれらのイベント内でのみ有効となり、イベントが終了すると解放されます。
