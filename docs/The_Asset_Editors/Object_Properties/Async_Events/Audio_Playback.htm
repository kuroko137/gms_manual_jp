Audio Playback

このイベントはオーディオキューの再生機能によってのみ作動し、async_load 変数に保存された DS マップを返します。この DS マップにはイベントを作動させたオーディオキューに関するキー / 値ペアが含まれています。イベントは再生するオーディオキューを指定し、そのオーディオキュー内のバッファが再生終了したときのコールバックによって作動します。

以下は DS マップにおいて利用可能なキーです。

  • "queue_id" - 再生を終了したキューのインデックスであり、audio_create_play_queue()によって返されます。
  • "buffer_id" - 再生されなくなったバッファの ID です。
  • "queue_shutdown" - 通常再生時は 0 になり、audio_free_play_queue()が呼び出されてイベントの受信が起きると 1 になります。1 になっている場合、キューに追加のデータを入れる必要はありません。

オーディオキューは複数のバッファから生成可能なため、場合によっては 1 つのキューに対してイベントが複数回、バッファリングされたオーディオの各領域の終わりに達するたびに作動することがあり、そのために "buffer_id" キーは用意されています。

注意: async_load 変数は Asynchronous イベント内でのみ有効となり、参照先となる DS マップはイベント開始時に自動で生成、終了時に破棄され、その他のときは変数の値が -1 にリセットされます。ただしイベント内で作成したほかのデータ構造については、適切な関数を用いて除去しなければなりません。

以下の例では 10 のバッファリングオーディオサウンドを追加したオーディオキューを生成し、キューを再生しています。

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);
for (var i = 0; i < 10; i++;)
   {
   audio_queue_sound(audio_queue, audio_buffer[i], 0, buffer_get_size(audio_buffer[i]));
   }
audio_play_sound(audio_queue, 0, true);

録音が検出されるとゲームステップごとに Asynchronous Audio Recording イベントが作動し、このように録音された入力を処理できます。

var queue = async_load[? "queue_id"];
var num = 0;
if queue = audio_queue
   {
   for (var i = 0; i < 10; i++;)
      {
      if async_load[? "buffer_id"] == audio_buffer[i]
         {
 buffer_seek(audio_buffer[i], buffer_seek_start, 0);
         num = i;
         }
      }
   if num == 9
      {
      audio_stop_sound(audio_queue);
      audio_free_play_queue(audio_queue);
      }
 }

ここではキュー ID をチェックし、必要なキューであればバッファ ID をチェックし、再生が終了したバッファを確認しています。さらにローカル変数に後でチェックするインデックス値を代入し、バッファをサウンドの開始位置に戻しています。そしてローカル変数が 9 になったらキューの終了を指示し、再生停止後にキューを解放します。
