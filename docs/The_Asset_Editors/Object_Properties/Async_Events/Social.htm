Social

このイベントはさまざまなソーシャル関数、および UWP Xbox Live 関数からのコールバックによって作動し、async_load 変数に保存される DS マップが返されます。この DS マップにはイベントを作動させた関数のコールバックに応じたキー / 値ペアが含まれています。

DS マップにはいくつかのキーがありますが、もっとも重要となるものが "id" キーです。このキーに返される定数をコードからチェックすることで、イベントが受信したコールバックを多くのコールバックの中から特定できます。キーを解析し、その値と有効な定数を比較すれば、残りの情報も取り出すことが可能です。

実績とリーダーボードの関数ページからはリファレンスを参照でき、イベントを作動させるための関数、コールバック、定数について詳しく調べることができます。なお次の場合は、ゲーム実行時に関数を呼び出さなくてもイベントが作動します。

  • OS の Dashboard からゲームを起動した場合
  • チャレンジを完了した場合
  • 別のプレイヤーがチャレンジを完了した場合
  • OS の Dashboard からチャレンジを選んでゲームを開始した場合

上記のコールバックから Social イベントが作動した場合、生成される async_load DS マップには以下の情報が含まれます（ "id" キーの値はイベントが呼び出された原因によって変化しますが、他の内容については同一となります）

  • "id" - イベントを作動させたコールバックのタイプに応じており、次の定数のいずれかが値となります。
定数 説明
achievement_challenge_received チャレンジを受信した
achievement_challenge_completed チャレンジを完了した
achievement_challenge_completed_by_remote 別のプレイヤーによってチャレンジが完了された
achievement_challenge_launched OS の Dashboard からチャレンジを選んでゲームを起動した

 

  • "playerid" - チャレンジをするプレイヤーの ID
  • "issuerid" - チャレンジを仕掛けてきた相手の ID
  • "state" - チャレンジの状態 （0 なら無効、1 なら保留、3 なら完了または拒否。値は文字列）
  • "message" - チャレンジのテキストメッセージ
  • "issueddate" - チャレンジの挑戦日時
  • "completeddate" - チャレンジの完了日時
  • "type" - 指定したチャレンジのタイプであり、次の 2 つの定数のいずれかとなります
定数 説明
achievement_type_score_challenge スコア値に基づくチャレンジ
achievement_type_achievement_challenge 実績に基づくチャレンジ

 

  • "identifier" - チャレンジの識別文字列
  • "score" - チャレンジと連動するスコア

以下は使用例となる部分的なコードです。

var ident = ds_map_find_value(async_load, "id" );
if ident == achievement_challenge_completed;
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
 state = ds_map_find_value(async_load, "state");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }
