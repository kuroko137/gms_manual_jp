Dialog

Dialog Asynchronous イベントは、専用の Asynchronous Dialog 関数からのコールバックが取得されたときにのみ作動します（ get_login_async()などの関数が該当し、この関数のページにはイベントの使い方に関するサンプルコードもあります）。

このイベントは名前、ログイン情報、番号、色などといったユーザー入力を要求します。多くのデバイスでは応答を待ち続けるループを受け付けないため、このようなイベントは非同期にしなくてはなりません。上記の関数によってダイアログが開かれている間、ユーザー入力によってイベントが作動するまで GameMaker Studio 2 はバックグラウンドで実行されます。つまりゲームの進行中にダイアログを開く場合、そのコールバックを受信するまでゲームを一時停止させる必要があります。

また、イベントが作動すると DS マップが返され、async_load という専用の変数にその ID が含まれます。マップに含まれる値はどの関数を利用したかによって異なります。詳しくはマニュアル内にある各関数のページを参照してください。
