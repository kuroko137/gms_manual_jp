Dialog

Dialog Asynchronous イベントは、get_login_async()といった専用の Asynchronous Dialog 関数からのコールバックが取得されたときにだけ作動します（イベントの使い方の詳細なコード例については関数のページを参照してください）。

これらのイベントは名前、ログイン情報、番号、色などのユーザー入力を要求するイベントです。ほとんどのデバイスでは応答を待つループを受け付けないためこうしたイベントは非同期にする必要があり、GameMaker Studio 2 ではこれらの関数によってダイアログが開かれている間、必要なユーザー入力を得てイベントが作動するまではバックグラウンドで実行されます。つまりゲームの進行中にダイアログが開かれている場合、ダイアログのコールバックを受信するまでゲームを一時停止するように対処しなければなりません。

同じく、async_load という専用の変数に保持される DS マップが返されます。このマップに保持されている値はどの関数を利用したかによって異なります。詳細については、マニュアル内にあるそれぞれの関数の記事を参照してください。
