Save / Load

ファイルへのバッファ読み込み・保存に関連する関数、およびメモリからオーディオをロード / アンロードしたときのコールバックによって作動するイベントです。このイベントには async_load という組み込み DS マップが追加され、特定の関数で必要となるキーが含まれます。それぞれ以下の項で紹介しましょう。

 

バッファ

buffer_save_async()または buffer_load_async()関数を呼び出すと、データの転送完了時に Asynchronous イベントが作動します。このイベントには、以下のキー / 値ペアからなる async_load マップが追加されます。

  • "id": 非同期 （Async）関数を呼び出したときに返される、その関数の ID です。
  • "status": データの保存 / 読み込みが正しく行われると true に、失敗すると false になります。

上記のキーによって保存 / 読み込みの進行状況を確認できるため、処理の完了時にメッセージを表示したり、ルームを切り替えるといったこともできます。

 

オーディオグループ

オーディオグループを扱う場合、audio_group_load()および audio_group_unload()関数によってメモリからオーディオグループをロード / アンロードできます。ロードする関数を呼び出すと、グループのオーディオファイル一式がメモリに読み込まれた時点でこのイベントが作動し、以下のキー / 値ペアからなるマップが追加されます。

  • "type": 呼び出されたイベントのタイプを示しており、オーディオを読み込んだ場合は "audiogroup_load" になります。
  • "group_id": 読み込まれたオーディオグループの ID （オーディオグループエディタで設定）を返します。

オーディオがすべてグループに読み込まれるとイベントが作動します。あとはルームを切り替えたり、BGM を再生するなど、自由に扱いましょう。
