Save / Load

ファイルへのバッファ読み込みと保存に関連する関数、およびメモリからオーディオをロードまたはアンロードしたときのコールバックによって作動するイベントです。このイベントには async_load という組み込み DS マップが含まれ、一部の関数に必要なキーが追加されます。以下はそうしたキーの一覧です。

 

バッファ

buffer_save_async()または buffer_load_async()関数を利用すると、データの転送完了時に Asynchronous イベントが作動します。このイベントには、以下のキー / 値ペアからなる async_load マップが追加されます。

  • "id": Async 関数を利用したときに返されるその関数の ID です。
  • "status": データの保存 / 読み込みが正しく行われると true に、失敗すると false になります。

これらのキーを利用すれば保存 / 読み込みの進行状況を問い合わせられるため、処理の完了時にメッセージを表示したり、ルームを切り替えるといったことができます。

 

オーディオグループ

オーディオグループでの動作時は、audio_group_load()と audio_group_unload()関数によってオーディオグループのロードとアンロードができます。ロード関数を利用した場合、グループのオーディオファイル一式がメモリに読み込まれるとこのイベントが作動し、以下のキー / 値ペアからなるマップが追加されます。

  • "type": 呼び出されたイベントのタイプを表しており、オーディオを読み込んだ場合は "audiogroup_load" になります。
  • "group_id": 読み込まれたオーディオグループの ID を返します（オーディオグループエディタで設定）。

すべてのオーディオがグループに読み込まれるとイベントが作動し、そこからルームを切り替えたり、BGM を再生したりすることができます。
