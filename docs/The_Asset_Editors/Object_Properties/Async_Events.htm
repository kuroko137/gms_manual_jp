Async イベント

Asynchronous イベントは、外部ソース （Web またはゲームを実行しているデバイス）からのコールバックを受信したときに起きるイベントです。たとえば画像の読み込みなどを GameMaker Studio 2 に行わせると処理の開始後、その間にしなければならない他の処理も続けて実行されます。処理のリクエストが完了するとコールバックが GameMaker Studio 2 に送信され、それに応じる Asynchronous イベントが起こります。

Keyboard イベントと同じように、Asynchronous Event イベントはすべてのインスタンスに対して起こります。たとえば http_get()をあるインスタンスで呼び出した場合、別のインスタンスの Asynchronous HTTP イベント内でそのコールバックを処理できます。

注意: ブラウザには XSS 保護があるため複数のドメインを経由するリソースの要求はブロックされ、Asynchronous の各イベントを利用したときに空の結果が返される場合があります。

Asynchronous イベントのカテゴリにはさまざまな種類のイベントがあり、それぞれ以下のページで説明されています。

注意: ページ内で挙げられている async_load 変数は Asynchronous イベントでのみ有効となります。参照先となる DS マップはイベント開始時に自動で生成、終了時に破棄され、それ以外の間は-1 に変数の値がリセットされます。
