Async イベント

Asynchronous イベントは、ゲームを実行しているデバイスまたは Web という外部ソースからのコールバックを受信したときに起きるイベントです。たとえば画像の読み込みなどを GameMaker Studio 2 に行わせるとその処理を開始しつつも、その間にしなければならない他の処理も続けられます。処理のリクエストが完了するとコールバックが GameMaker Studio 2 に送信され、それに応じる Asynchronous イベントが起こります。

Asynchronous Event イベントは Keyboard イベントと同じように、すべてのインスタンスに対して起こります。たとえば http_get()をあるインスタンスで呼び出した場合、別のインスタンスの Asynchronous HTTP イベント内でそのコールバックを処理させることが可能です。

注意: ブラウザには XSS 保護があるため複数のドメインを経たリソースの要求はブロックされ、以下のイベントを利用したときに空の結果が返される場合があります。

Asynchronous イベントのカテゴリにはさまざまな種類のイベントが含められており、それぞれ以下の項で説明されています。

注意: 以下の項で取り上げている async_load 変数は、Asynchronous イベントでのみ有効となります。参照先となる DS マップはイベント開始時に自動で生成、終了時に破棄され、その他のときは変数の値が -1 にリセットされます。
