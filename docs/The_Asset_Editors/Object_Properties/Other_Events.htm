Other イベント

このページでは、オブジェクトエディタの Other イベントカテゴリにまとまっている各イベントについて解説されています。

 

Outside RoomOutside Room 

このイベントは、インスタンスがルームの外へ出たときに作動します。インスタンスに割り当てられたスプライト（およびスプライトの設定値）から判定され、画像の x / y スケールを 1 以上にした場合では、スプライト全体が画面外に出るまで作動しません。スプライトがない場合はインスタンスの位置から判定され、x / y 位置がルームの境界外となった瞬間に作動します。一般的には敵の弾などに利用されるイベントです。ルームから出た弾を破棄しておけば、何百万もの弾が飛び続け、ゲームを重くするようなこともありません。なお、このイベントはインスタンスが初めてルームを出たときにだけ作動するためご注意ください。

 

Intersect BoundaryIntersect Boundary

このイベントは、インスタンスがルームの内側の端に「接触」したときに作動します。Outside Room イベントと同じく、インスタンスに割り当てられているスプライトから判定されるため、画像の x / y スケールを 1 以上にした場合、スプライトの一部がルームの境界と交差した瞬間に作動します。スプライトがない場合はインスタンスの位置から判定され、x / y の位置がルームの境界と交差した瞬間にイベントが作動します。このイベントを利用することで、インスタンスをルームの外に出さずに「跳ね返らせる」といったことができます。

 

ViewsViews

カメラのビューは通常ではルームエディタから定義し、広いルームの一部範囲を表示するためのものです。このイベントカテゴリは 2 つのサブカテゴリ（ Outside View と Intersect Boundary ）に分かれており、利用可能な 8 つのビューポートごとにイベントが存在します。カテゴリがもつ機能は各ルームイベントと同じであり、違いはルームではなく、カメラビューの境界をインスタンスに判定させることだけです。

 

Game StartGame Start

特殊なイベントであり、ゲームを通して一度だけ、このイベントにコードまたはアクションが追加されているインスタンスにおいて作動します。コードまたはアクションから動的に生成されたインスタンスではなく、ルームエディタからゲームの最初のルームに配置されたインスタンスでなければなりません。このイベントは各インスタンスの Create イベントよりも後に起こるため、Create イベントで事前に定義された変数をコードまたはアクションに含めることができます。このイベントは一般的には「制御用」のオブジェクトに定義し、グローバル変数の初期化、BGM の開始、ファイルの読み込みといったゲーム開始から一度だけ実行するような処理を行わせるためだけに利用します。

 

Game EndGame End

上記の Game Start イベントと似ていますが、このイベントはゲームを通して一度だけ、ゲームのウィンドウが閉じられる直前に作動します。そして、ゲームが閉じられた時点のルームにいるインスタンスに対してのみ有効となります。注意点として、このイベントは Windows 、Ubuntu(Linux) 、macOS でのみ実行可能です。これはモバイルデバイスではこのイベントが GameMaker Studio 2 によって呼び出される前にゲームが終了してしまう場合があるためです。HTML5 モジュールによって作られたブラウザゲームでは、ゲームのタブまたはブラウザを閉じたときに動作しますが、すべてのブラウザでサポートされている機能ではないため（解説時点では Firefox 、Chrome 、Safari のみ対応しています）、利用する場合は気をつけてください。このイベントは一般的にデータを保存するために使われます。

 

Room StartRoom Start

このイベントは、最初からルームにいるインスタンスに対してルーム開始時に起こります。Create イベントよりも後に起こるため、Create イベントで定義された変数などを利用することが可能です。

 

Room EndRoom End

This event happens to all instances that are in the room when the room ends. This event is very useful for "cleaning up" after a level has ended, for example you can delete loaded resources here, or remove a particle system to prevent memory leaks etc...

 

Animation EndAnimation End

In GameMaker Studio 2 sprites are not static things as they can have sub-images and be animated at different speeds. Each frame of an animation (called a sub-image) has its own number, starting at 0, which can be checked in code or through actions, but sometimes all you really need to know is when the animation has ended. That's when this event is triggered, right at the end of the animation when the sub image index shows that the last frame has been reached. This event is really useful for many things, for example an explosion object where you can set the instance to destroy itself when the last frame of the animation is reached.

 

Animation UpdateAnimation Update

This event is designed only for use with the Skeletal Animation Functions. It is a special event that is triggered every step in an instance that uses a skeletal animation sprite, and is designed to "intercept" the bone data after the orientation of the bones has been calculated for the current animation state but before this data is committed to use for drawing. This allows you to make modifications to the bone data using the appropriate functions.

 

Animation EventAnimation Event

This event is designed only for use with the Skeletal Animation Functions. This event is triggered in an object which has a skeletal animation sprite assigned to it, and where the object is using the default draw for the assigned sprite, or the sprite is being drawn via draw_self(). It currently won't trigger on skeleton animation sprites drawn any other way. The event is designed to capture and transmit data that has been assigned to particular events in the sprite animation as defined in the editor used to make them.

When the Animation Event is triggered, a special DS Map will be created called the event_data map (much the same as we have the async_load map for an async event). This map will have all of the following key/value pairs that you can parse to get information about the event:

  • "name": the name of the event (as defined in the animation program)
  • "track": the index of the track the animation is playing on (default is 0)
  • "integer": an integer value associated with the event (as defined in the animation program, default is 0)
  • "float": a float value associated with the event (as defined in the animation program, default is 0.0)
  • "string": a string value associated with the event (as defined in the animation program, default is an empty string "")

 

Path EndedPath Ended

Paths can be an important part of making any game, whether it is for a tower defence type game, or a shoot-em-up or whatever, and it's often really important that an instance does some particular action when it reaches the end of its path. This event detects this and is triggered when it happens allowing you to add in code or actions to deal with the circumstances, for example in a tower defence game you would use this to destroy the instance and remove a health point from the goal object.

 

User EventsUser Events

These are special events that are not triggered by GameMaker Studio 2 automatically, but have to be implicitly called by you using code or actions while the game is running. There are 12 user events in total and they can contain code or actions and they can be called at any time in the game loop, making them very useful for creating your own events that happen when you decide.

 

Broadcast MessageBroadcast Message

This event is triggered whenever a sprite or a sequence generates a broadcast message string. For more information please see the page on Broadcast Messages.
