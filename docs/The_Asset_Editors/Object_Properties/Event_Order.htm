イベントの順番

GameMaker Studio 2 のイベントについて考える上で覚えておきたい点は、各ステップにおいて発生するイベントの順番は明示できないということです。なぜならこれは GameMaker Studio 2 の内部構造に基づいているため、ソフトウェアの開発によって変更させる可能性があるからです。ただし一部のイベントに関しては、必ず同じ順番で実行されます。

必ず同じように発生するものとして、まずはルームに初めて入ったときに起きる一連のイベントがあります。これらのイベントは以下の順番で引き起こされます。

  • オブジェクト変数 / インスタンス変数は初期設定されます（これらはオブジェクトエディタの IDE 、およびルームエディタから定義できる変数です）。変数付きのオブジェクトがある場合、他のものよりも先にそれらのオブジェクトが生成され、変数付きのインスタンスはその次に定義されます。よってオブジェクト変数は、ルームエディタからインスタンスごとの変数で上書きすることが可能です。
  • Create イベントに続いて各インスタンスの生成コードを実行 - インスタンス生成時、まずは Create イベントを実行し、それからインスタンスの生成コードを実行して次のインスタンスの生成に移ります。
  • Game Start イベント - このイベントはゲーム開始直後のルームで、ルームエディタから配置したすべてのインスタンスにおいて作動します （game_restart() イベントを呼び出すともう一度作動します）。
  • ルームの生成コード - ルームに最初に入ったときに一度だけ実行されるコードであり、ルームエディタから記入できます。
  • インスタンスの Room Start イベント - "Other" カテゴリのイベントであり、永続的かどうかを区別せず、すべてのインスタンスで作動します。

もう一つ覚えておきたい点として、各インスタンスが生成される順番はルームエディタを開き、インスタンスレイヤーのプロパティウィンドウに一覧表示されるインスタンスを上下に移動させると変更できます。

上記のイベント以外にも、3 つの Step イベント、各種の Draw イベントはどのようなオブジェクトであっても必ず同じ順番で発生します。順番はずっと変わらないため、ゲームの各ステップ間に決められたタイミングで実行しなければならないコードがある場合、以下のイベントを利用するといいでしょう。

  • Begin Step イベント
  • Step Event (この Step イベントはインスタンスが新しい場所に配置される直前に実行されます)
  • End Step イベント

描画用の各イベントも決められた順番で処理されます（ただし Window Resize イベントは例外であり、異なる形で作動します）。

  • Pre Draw イベント
  • Draw Begin イベント
  • Draw イベント
  • Draw End イベント
  • Post Draw イベント
  • Draw GUI Begin イベント
  • Draw GUI イベント
  • Draw GUI End イベント
