スクリプトエディタ

ここではスクリプトエディタ（テキストエディタとも呼びます）について扱います。このエディタにはゲームを動かすためのコードを入力することになり、スクリプトアセットの作成、オブジェクトイベントのコーディング、ルームの生成コード追加など、さまざまな場面で呼び出されます。ここではスクリプトエディタの仕組みを簡単に紹介するに留め、スクリプトアセットの作り方や、Drag and Drop™ によるスクリプト構築については取り上げません。この 2 つについては以下を参照してください。 

  • GML スクリプト
  • DnD™ スクリプト

スクリプトエディタには、GameMaker Studio 2 の内蔵プログラミング言語である GameMaker Language のコードを書くことができます（詳しくは GML リファレンスを参照してください）。GameMaker Studio 2 に慣れてきて、最大限まで使いこなしたくなったら、ゲーム制作の可能性を広げるためにもこの言語を習得することをお勧めします。

スクリプトアセットは名前を変更でき、アセットブラウザに並んでいるアセットを右クリックし、「名前変更」を選ぶと別の名前にできます。ただし関数のスクリプティング規則を守る必要があり、利用できるのは英数字のみ、最初の 1 文字は英字にしなければならず、入れられる記号は "_" （アンダーバー）のみとなります。スクリプトエディタが他の形で開かれている場合は変更できず、オブジェクトイベントなどといった編集対象の名前が入ります。

スクリプトエディタのウィンドウが開かれると、以下のオプション / レイアウトが表示されます。

スクリプト / イベントタブスクリプト / イベントタブ

スクリプトエディタの上部にはタブが並んでおり、複数のスクリプトおよびイベントを 1 つのウィンドウで扱うことができます（この動作は環境設定から変更でき、スクリプトおよびイベント別にウィンドウを開くこともできます）。タブはドラッグして並び替えできるほか、ワークスペース内にドラッグすると新しいウィンドウが開きます（別のウィンドウに入れることも可能です）。また、エディタを最大化すると新しいワークスペースが開きます。タブは IDE のウィンドウからドラッグして出すことも可能で、メインウィンドウと同じように扱える新しい IDE を開いてスクリプトリソースを編集できます。

オブジェクトイベントのコードを編集しているとき、別のウィンドウ、ワークスペースに分離するか最大化してからエディタ上を右クリックすると、メニューに以下の項目が現れます。

  • ワークスペースに復元: スクリプトエディタの最大化 / ワークスペース状態を解除し、ワークスペースの元オブジェクトに繋ぎ直します。
  • オブジェクトに移動: ワークスペースを開き、スクリプトの元オブジェクトに移動します。
  • イベントを追加 / 開く: スクリプトの元オブジェクトに新しいイベントを追加し、そのイベントを別のタブで開きます。イベントにコードがすでにある場合、別のタブで開くだけとなります。

右クリックメニューの他の項目については、ページの下部でリンクされている編集のページを参照してください。

 

余白余白

余白にはコードの行番号や、コードの状態をあらわす情報が表示されます。特に重要なものとして、コードの構成ミスやタイポがある場合、IDE によってそのコード行に構文エラーを示すマークが表示されます。マークの上にカーソルを置くと、どのような問題があるのか簡単にわかります。

構文エラー、またコードのテスト、コンパイル時に起こる一般的なエラーについての詳細は、以下のページを参照してください。

  • エラー報告

他にも、ブレークポイントが置かれている行には目印が表示されます。ブレークポイントとは、デバッガにゲームの実行を一時停止させる位置のことです。ブレークポイントはスクリプト、オブジェクトイベントの各行に設定でき、"F9" を押すか右クリックメニューから配置を切り替えることができます。ブレークポイントを置くと、余白にはこのように表示されます。

余白にはブックマークされた項目も表示されます。行をブックマークするには / + + 数字キー （0 から 9）を同時に押しましょう。ブックマークすると、 / + 数字キーを押してその行までジャンプできるようになります。ブックマークを削除したくなったら、行にカーソルを合わせて追加時と同じ操作をしてください。ブックマークがある場合はこのように表示されます。

 

検索 / 置換検索 / 置換

スクリプトエディタでの作業時は、 / + "F" で検索ボックス、 / + "H" で検索と置換ボックスを開くことができます。

ここにキーワードを入力するとローカル検索ができ、ボックスの右上にある矢印をクリックすると同じスクリプト内の該当する語句まで順番に移動できます。以下のボタンをクリックすると検索方法を変更可能です。

オンにすると、入力した文字列全体とマッチする語句だけが強調表示されるようになります。たとえばオフにして "random" で検索すると、irandom()、randomise()といった同じ文字列を含むすべての語句が候補となりますが、オンにした場合は random()のような語句でなければ候補となりません。
検索語の内容に加えて、大文字と小文字をチェックするオプションです。たとえば "spr_Dog" というスプライトを "dog" という語句で検索した場合、オフなら強調表示されるものの、オンにしている場合は "dog" と "Dog" は別の文字列として扱われ、強調表示されません。

検索ウィンドウを / + "F" で開くと置換機能は表示されないため、置換ボタンをクリックして開く必要があります。ボックスが開くと置換語を入力できるようになり、以下のボタンから置換を実行できます。

クリックすると、検索して見つかった次の文字列を置換語で置き換えます。次というのはカーソル位置より下にある文字列のことで、検索ボックスの右上にある矢印ボタンをクリックするとそれぞれの文字列に移動できます。
クリックすると、マッチした文字列すべてを置換語で置き換えます。

なお、 / + + "F" でグローバル検索が可能です（現在のスクリプト、イベント内だけでなくプロジェクト全体を検索します）。

メインコードエディタは、スクリプトアセットやオブジェクトイベントに必要なコードを書くための領域です。コードの具体的な書き方については、以下にリンクされている編集のページで取り上げられています。

 

情報情報

IDE の下部にある情報バーには、現在の行番号、行内での位置が表示されます。コードヘルパーも表示され、編集中の関数とその引数を確認できます。なお、関数の入力時はコードヘルパーに編集中の引数が強調表示されます。JSDoc スクリプトコメントを利用しているスクリプトアセットの場合、提示した情報も表示されます。
