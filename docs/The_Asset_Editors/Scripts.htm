スクリプトエディタ

ここではスクリプトエディタ（テキストエディタとも）について扱います。このエディタには、ゲームを動かすためのコードを入力できます。スクリプトエディタは専用のスクリプトアセットを作成したり、オブジェクトイベントのコーディングや、ルームの生成コード追加など、さまざまな事ができます。ここではスクリプトエディタの仕組みを簡単に紹介するに留め、スクリプトアセットの作成方法や、Drag and Drop™ によるスクリプトの作り方は取り上げません。これらについては、以下のページを参照してください。 

  • GML スクリプト
  • DnD™ スクリプト

スクリプトエディタは、GameMaker Studio 2 の内蔵プログラミング言語である GameMaker Language のコードを書くためのものです（詳しくは GML リファレンスを参照してください）。GameMaker Studio 2 に慣れてきて、最大限まで使いこなしたくなったら、ゲーム制作の可能性を広げるためにもこの言語を習得することをお勧めします。

アセットブラウザのスクリプトアセットを右クリックし、名前変更を選ぶと新しい名前にできますが、スクリプトの名前は関数のスクリプト規則に合わせる必要があり、利用できるのは英数字のみ、最初の一文字は英字にし、記号として加えられるのは "_" 記号のみとなります。スクリプトエディタが他の形で開かれていると名前は変更できず、オブジェクトイベントなど、編集元の名前となります。

スクリプトエディタのウィンドウが開かれると、以下のオプションおよびレイアウトが与えられます。

スクリプト / イベントタブスクリプト / イベントタブ

スクリプトエディタが開かれるウィンドウの一番上にはタブが並んでおり、複数のスクリプトおよびイベントを 1 つのウィンドウで扱うことができます（この動作は環境設定から変更でき、スクリプトおよびイベントごとにウィンドウを開くことも可能です）。タブはドラッグして並びを変更したり、外にドラッグしてワークスペースに新しいウィンドウとして開けるほか、スクリプトエディタを新しいワークスペースに広げることも可能です。タブを IDE のウィンドウからドラッグして出すと、スクリプトリソースが新しい IDE で開かれ、メインウィンドウと同じように扱うことができます。

スクリプトエディタからオブジェクトイベントのコードを編集している場合、エディタを最大化または別のウィンドウ、ワークスペースに分離してからエディタを右クリックすると、このように追加の項目が表示されます。

  • ワークスペースに復元: スクリプトエディタの最大化 / ワークスペース化を解除し、ワークスペースのオブジェクトに繋ぎ直します。
  • オブジェクトに移動: コードの元オブジェクトがあるワークスペースを開き、そのオブジェクトにフォーカスします。
  • イベントを追加 / 開く: 現在のスクリプトを有しているオブジェクトに新しいイベントを追加し、スクリプトエディタの新しいタブに追加したイベントを開きます。選択したイベントにコードがすでにある場合、イベントを新しいタブで開きます。

他の右クリックメニュー項目については、以下にリンクされている編集のページを参照してください。

 

余白余白

余白の部分にはコードの行番号や、コードの状態をあらわす特定の情報が表示されます。特に重要なものとして、タイポやコードの構造に誤りがある場合、IDE によってコード行に構文エラーを示す赤い！マークが表示され、問題を知らせてくれます。このマークの上にマウスを置くと、どのような問題があるのか簡単に説明してくれます。

構文エラー、およびコードのテスト、コンパイル時に起こる一般的なエラーについては以下のページに詳細があります。

  • エラー報告

また、余白部分にはブレークポイントが追加されている行に目印がつけられます。ブレークポイントとは、スクリプト内の追加位置に達したとき、デバッガにゲームの実行を一時停止させるための地点です。ブレークポイントはスクリプト、オブジェクトイベントの各行に追加でき、"F9" を押すか右クリックメニューから切り替えることができます。ブレークポイントを追加すると、余白部分にはこのように表示されます。

最後に、余白にはブックマークされた項目も表示されます。コード行をブックマークするには / + + 数字キー （0 から 9）を同時に押しましょう。行をブックマークすると、 / + 数字キー （0 から 9）を押すことで IDE のどこからでもその行まで移動できるようになります。ブックマークを削除したくなったら、同じ操作を追加した行で行ってください。ブックマークを追加するとこのように表示されます。

 

検索 / 置換検索 / 置換

スクリプトエディタでの作業時は、 / + "F" で検索ボックス、 / + "H" で検索と置換ボックスを開くことができます。

ここでは入力したキーワードでローカル検索を行うことができます。検索語を入力し、検索ウィンドウの右上にある矢印をクリックすると同じスクリプト内の該当する語句まで順番に移動できます。また、以下のボタンから検索方法を変更することも可能です。

有効にすると、入力した文字列全体とマッチする文字列だけが強調表示されるようになります。たとえば無効のまま "random" で検索すると、irandom()、randomise()といった同じ文字列を含むすべての単語が候補となりますが、有効にしてから検索すると、random()関数だけが候補となります。
このオプションを有効にすると、検索する文字列の内容だけでなく、大文字か小文字かどうかもチェックされるようになります。たとえば "spr_Dog" というスプライトを "dog" という語句で検索した場合、オプションが無効だとスプライトの文字列が強調表示されますが、有効にしている場合、"dog" と "Dog" が同じものとして見なされなくなるため強調表示されません。

検索ウィンドウを / + "F" で開いた場合、置換機能は表示されないため置換ボタンをクリックして開かなくてはなりません。ウィンドウが開いたら、検索した文字列を置き換えるための文字列を入力しましょう。以下のボタンから置換を実行できます。

このボタンをクリックすると、同じスクリプト内で見つかった次の検索文字列を置換文字列で置き換えます。「次」というのは現在のカーソル位置より下にある検索文字列のことで、検索ウィンドウの右上にある矢印ボタンをクリックするとそれぞれの文字列に移動できます。
このボタンをクリックすると、取り出されたすべての検索文字列を置換文字列で置き換えます。

なお、 / + + "F" を押すとグローバル検索を行うことができます（現在のスクリプト・イベント内だけでなくプロジェクト全体を検索します）。

メインコードエディタでは、スクリプトアセット作成やオブジェクトイベントに必要なコードを書くことになります。コードの具体的な編集方法については、以下にリンクされている編集のページで取り上げられています。

 

情報情報

IDE の下部にある情報バーには現在の行番号、行内での位置が表示されます。またコードヘルパーも表示され、編集中の関数、および必要となる引数が 1 行のテキストとして表示されます。エディタに関数の内容をすべて入力すると、編集中の引数が強調表示されるようになります。スクリプトアセットで JSDoc スクリプトコメントを利用している場合、与えた情報も表示されます。
