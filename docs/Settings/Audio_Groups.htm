オーディオグループ

オーディオグループマネージャーは IDE のツールメニューから呼び出すことができます。ここではオーディオグループの追加、削除、名前変更ができ、出力するプラットフォームを設定できます。GameMaker Studio 2 ではオーディオリソース（効果音および BGM）にさまざまなオーディオグループを割り当てることで、同時に再生されるサウンドの数を最適化したり、プラットフォームごとにより詳細な制御を可能とします。

オーディオグループを定義するには、新規追加ボタンをクリックして新しいグループを作成し、名前を入力しましょう。グループを切り替えたくなったら、ドロップダウンメニューをクリックすると編集グループを選択できます。

注意: サウンドエディタからストリーミングオーディオとして指定している場合、オーディオグループを割り当てることはできません。ストリーミングされるサウンドは他のサウンドと同じようにパッケージ化されず、読み込み、再生、読み込み破棄されるタイミングはコードから制御することになります。

サウンドをオーディオグループに追加するには、サウンドエディタで開いてグループを選ぶか、アセットブラウザに表示されているサウンドを右クリックし、オーディオグループに追加しましょう。サウンドアセットは複数選択し（またはグループフォルダを選択）、右クリックメニューからオーディオグループに追加することも可能です。また、オーディオグループエディタの一番下にあるリソース追加ボタンをクリックしてもアセットを追加できます。

サウンドを作成したオーディオグループに追加すると、出力するプラットフォームを指定できるようになります。どのサウンドも標準では高品質に設定されていることでしょうが、HTML5 などでは低品質にしてメモリを削減しなくてはならないかもしれません。そのようなときは低品質のファイルを追加し、別のオーディオグループに割り当ててみてください。それからオーディオグループエディタを開き、そのグループを選んで HTML5 にのみ出力するよう設定し、さらに高品質のオーディオグループから HTML5 を除外するといいでしょう。注意点として、デフォルト （default）のオーディオグループは出力設定を変更できないため、ゲームの最終パッケージビルド時にはすべてのプラットフォームにアセットを出力します。

オーディオグループを定義してからサウンドを割り当てると、グループ内のサウンドが左側に一覧表示されます。デフォルトのオーディオグループは必ず有効となるほか、含まれているサウンドはすべてのプラットフォームで出力され、起動時にメモリへ読み込まれます（サウンドエディタのプロパティから「ストリーミング」に設定している場合は異なります）。それとは逆に、作成したオーディオグループ内のファイルは audio_group_load()関数を呼び出すまでメモリに読み込まれないため、メモリに常駐させるオーディオを管理することが可能です。なお、オーディオグループ関数ではオーディオグループの ID 値を指定しなくてはなりませんが、オーディオグループの名前がそのまま ID となります。

オーディオグループは構成とも関連しています。構成を選択してからオーディオグループを開くと、ウィンドウの右側からその構成における出力ターゲットを指定でき、別の構成に変えるとそれぞれの出力ターゲットも切り替わります。しかしながら、サウンドリソースを構成ごとに異なるグループに割り当てることはできません。指定できるのはオーディオグループの出力ターゲットのみです。
