オーディオグループ

オーディオグループマネージャーは IDE のツールメニューから呼び出すことができます。ここではオーディオグループの追加、削除、名前変更ができ、出力するプラットフォームを設定できます。GameMaker Studio 2 ではオーディオリソース（効果音および BGM）にさまざまなオーディオグループを割り当てることで、同時に再生されるサウンドの数を最適化したり、プラットフォームごとにより詳細な制御を可能とします。

オーディオグループを定義するには、新規追加ボタンをクリックして新しいグループを作成し、名前を入力しましょう。グループを切り替えたくなったら、ドロップダウンメニューをクリックすると編集グループを選択できます。

注意: サウンドエディタからストリーミングオーディオとして指定している場合、オーディオグループを割り当てることはできません。ストリーミングされるサウンドは他のサウンドと同じようにパッケージ化されず、読み込み、再生、読み込み破棄されるタイミングはコードから制御することになります。

サウンドをオーディオグループに追加するには、サウンドエディタでサウンドを開いてオプションからグループを指定するか、アセットブラウザで右クリックし、オーディオグループに追加しましょう。サウンドアセットは複数選択し（またはグループフォルダを選択）、右クリックメニューからオーディオグループに追加することも可能です。また、オーディオグループエディタの一番下にあるリソース追加ボタンをクリックしてもアセットを追加できます。

サウンドを専用のオーディオグループに追加できたら、どのプラットフォームに出力するかを同じウィンドウで選択できます。デフォルトではすべてのサウンドが高品質にされていますが、HTML5 などでは低品質にしてメモリを削減する必要があるかもしれません。そういった場合は低品質のファイルを追加し、別のオーディオグループに割り当てるといいでしょう。あとはそのグループをオーディオグループエディタで選択し、HTML5 にのみ出力するよう設定し、高品質のオーディオグループから HTML5 を外すだけです。重要な点として、「デフォルト」のオーディオグループは出力設定を変更できないため、ゲームの最終パッケージのビルド時には必ずすべてのプラットフォームに出力されます。

オーディオグループを定義してからサウンドを割り当てると、グループにあるサウンドが左側に一覧表示されます。「デフォルト」のオーディオグループは常に利用でき、このグループに存在するサウンドは必ずすべてのプラットフォームのゲームに含まれ、起動時にメモリへ読み込まれます（サウンドエディタのプロパティから「ストリーミング」に設定していない場合）。いっぽうで、カスタムオーディオグループに存在するファイルは、audio_group_load()関数を呼び出すまでメモリに読み込まれません。よってメモリに常駐させるオーディオを管理することが可能です。なおオーディオグループ関数には、オーディオグループの ID 値を指定しなくてはなりませんが、この ID はオーディオグループの名前となっています。

オーディオグループは構成とも結びついています。構成を選んでからオーディオグループのウィンドウを開くと右側からその構成用の出力ターゲットを指定でき、構成を変えた場合はそれぞれの設定も切り替わります。なお、サウンドリソースを構成ごとに異なるグループに割り当てることは不可能です。グループの出力ターゲットのみ指定できます。
