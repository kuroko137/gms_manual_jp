テクスチャグループ

テクスチャグループマネージャーは IDE のツールメニューから開くことができ、テクスチャグループの追加、削除、名前変更ができます。GameMaker Studio 2 では画像アセット（スプライト、タイルセット、フォント）にテクスチャグループを割り当て、ハードウェアによって任意のタイミングで実行されるゲーム中のテクスチャスワップ（切り替え）回数を最適化することが可能です。このように機能させるには、まずマネージャーの「新規追加」ボタンをクリックして必要なテクスチャグループを定義し、それから各画像アセットを開いてドロップダウンメニューから所属グループを指定しなくてはなりません。また、アセットブラウザの右クリックメニューからテクスチャグループに追加することも可能で、複数のアセット（またはグループフォルダ）を選択してからメニューを開いた場合、アセットをまとめてテクスチャグループに追加できます。以下の画像では、タイルセットをまとめて追加しようとしています。

テクスチャグループには複数のテクスチャページを追加できますが、テクスチャグループとページが異なるものだということは覚えておきましょう。たとえばレベル（ステージ）ごとに専用の画像が多数ある場合、テクスチャグループをレベルごとに割り当てれば良いのですが、各画像には最低でも 2 つのテクスチャページがグループ単位で必要となります。

ゲームのあらゆる画像は、デフォルトでは必ず存在する "Default" テクスチャグループに追加されますが、新規追加ボタンをクリックすると新しいテクスチャグループを作成できます（名前の部分を選ぶことで好きな名前を再入力できます）。グループを作成した直後はスプライト、タイルセット、フォント部分に「このグループは空です」というメッセージが表示されることになり、各画像アセットを開いて追加しなくてはなりません。一番下のリソースを追加ボタンをクリックしてリソースを追加するか、画像、タイルセット、フォントの各エディタからテクスチャグループを選択しましょう。Default 以外のテクスチャグループは自由に削除可能で、削除するとグループ内のリソースはすべて Default グループに移されます（間違ってグループを削除してしまった場合は、 / + "Z" で取り消すことができます）。

テクスチャグループを定義して画像アセットを割り当てると、テクスチャグループエディタの左側にアセットが一覧表示され、そのうちのスプライトをクリックした場合は右側にプレビューが表示されます。大規模なプロジェクトでグループに割り当てられている画像を確認したいときに役立つでしょう。割り当てられている画像に複数のサブ画像がある（アニメーションする）場合、プレビューウィンドウには最初のサブ画像のみ表示されるためご注意ください。また、リソースを右クリックするとメニューが表示され、別のテクスチャグループに移したり、対応するエディタで開くことができます。

ここではスプライト、タイルセット、フォントといったアセットがそれぞれ別々のものとして並べられていますが、実際にはすべて同じテクスチャページに追加されることになり、リソースの種類ごとにページが分けられることはありません。ただし「テクスチャページを分ける」としてチェックされているスプライトアセットは例外となり、そのスプライト専用のテクスチャページが用意されます（詳しくはスプライトエディタのページから「テクスチャ設定」を参照してください）。

テクスチャページの役割や、どのように生成されるかについては以下のページで詳しく解説されています。

  • テクスチャページ

テクスチャグループエディタのグループ設定からは、以下のオプションによって指定したグループのテクスチャページの生成方法を正確に決めることができます。

  • 拡大縮小の許可 - チェックを外すと、いかなる場面でもこのテクスチャグループに拡大縮小が適用されなくなります。ユニバーサルアプリ（複数のデバイスに対応するアプリケーション）を扱うにあたって特に役立つオプションで、フォントなどの一部リソースを専用のグループに入れ、アプリを特定のデバイスで実行したときに利用することができます。たとえば Retina ディスプレイでは綺麗に表示されるものの、通常のディスプレイでは見た目が悪くなってしまうフォントがある場合、拡大縮小しない専用のグループに割り当てることができます。もし iPhone 向けに開発している場合、テクスチャが拡大縮小されると画像がぼやけるせいでアプリが拒絶されるため、チェックを入れておくべきでしょう。デフォルトでは有効としてチェックされています。
  • 自動クロップ - GameMaker Studio 2 に画像が追加された際、画像の周りに余白（完全な透明部分）があり、なおかつ「自動クロップ」がチェックされている場合はその余白を自動的に切り取ります。ただし、画像がプリミティブなどを対象とする繰り返しされないテクスチャであった場合、テクスチャマッピングの UV 座標が正確に再現されなくなってしまいます。このような場合はチェックを外しましょう。スプライトはテクスチャページに「元のまま」追加され、周囲の透明部分も残されます。デフォルトでは有効としてチェックされています。
  • ミップマップを生成 - チェックすると（デフォルトでは無効です）、元のテクスチャページを拡大縮小し、テクスチャファイル内に複数の解像度でフィルタリングされるようになります。通常は無効にしますが、ルームの広い領域でカメラのズーム倍率を変更したり、3D のゲームを制作しているなら、有効にするとミップマップ関数を利用し、各段階のミップマップをゲームがどのように処理するかを調整しなくてはならないでしょう。チェックすると、同時に「境界のサイズ」が 8 に変更されます（元から 8 未満の場合）。これはミップマップを使用すると、テクスチャページの画像間により広い境界が必要となり、境界が狭いとミップ段階が高くなったときににじみが生じてしまうということです。デフォルト値の 8 なら、サブ画像がにじみ始める前に画像を元のサイズの約 1/8 まで縮小できることになりますが、この値は必要に応じて変更可能です。なお、スプライトの「テクスチャページを分ける」オプションを利用している場合、各テクスチャにチェックを入れてミップマップを生成させる必要はなく、一般ゲームオプションを開いて「分けたテクスチャページのミップマップを生成」にチェックを入れるといいでしょう。
  • 境界のサイズ - 境界のサイズによって、テクスチャグループに生成されたページ上にある各アセットの外周部の重複（もとい「ピクセル倍化」）量を変更でき、画像が拡大縮小された際、間に余白（あるいはつなぎ目）が表示されなくなります。デフォルトでは 2 ピクセルですが、ビューのズーム倍率を大幅に変更する極端な場合では、値を増やさなくてはなりません。なお、タイルセットエディタにはタイルの出力境界を設定できるオプションがありますが（なぜ必要かも説明されています）、ここでの境界設定とは異なるものであり、タイルは一時テクスチャにタイルごとの出力境界を加えて生成され、それからテクスチャページに指定した外周部分の境界を加えて追加されます。また、ミップマップを有効にするとテクスチャページのデフォルト境界サイズが自動的に 8 に設定され、高いミップ段階での「にじみ」を防ぐことができますが、別の値に変更することも可能です。
  • 親テクスチャグループ - 現在のグループの「親」とする既存のグループを指定します。「子」のグループは親テクスチャグループに加えられた変更をすべて引き継ぎ、主にクロスプラットフォームのゲームを扱う場合のリソース管理をするためのものです。テクスチャグループは親を指定しても独立したグループであり、他のグループと同じようにテクスチャページを生成しますが、親を割り当てることでプラットフォームごとに大量のテクスチャグループを除外できます。たとえば "player" グループに Android 用のプレイヤースプライトを大量に追加、Android 用の敵スプライトを "enemy" グループに追加し、両方に "android" という親グループ割り当てるとします。そして Windows や他のプラットフォームのビルドでは "android" グループを除外すると、子グループも同じように除外されます。

これらの設定以外にも、グループの使用先に表示されている各チェックボックスからテクスチャグループの出力プラットフォームを切り替えることができます。ただしテクスチャグループは必ずすべてのプラットフォームに出力されるため、チェックを付け外しすることはできません（カスタムテクスチャグループのみ可）。

テクスチャグループは構成とも関連しています。具体的には、それぞれの構成用のテクスチャグループを定義し、スプライト、タイルセット、フォントリソースの割り当てを各グループで行うことで、別の構成に切り替えても割り当てが「記憶」されるということになります。テクスチャグループのウィンドウ内にある出力オプションも同じであり、構成ごとにテクスチャグループの出力プラットフォームを設定できます。

使い方の一例として、iPad の構成で 2048x2048px のテクスチャページを作成するとしましょう（ゲームオプションから設定を行います）。それから各画像アセットを適切なテクスチャグループに割り当てたのち、構成を変更します（たとえば HTML5 に）。そしてテクスチャグループのウィンドウから、512x512px のテクスチャグループを新しく作成し、アセットを割り当てます。また、ページの下部にはテクスチャグループ設定を別の構成にコピーするボタンもあります。

最後に、テクスチャグループはビットマップ画像にのみ関連し、非ビットマップスプライトはテクスチャグループに割り当てることができません。あらかじめ専用のテクスチャページが用意されるスプライトもありますが（スケルタルアニメーションスプライト）、それ以外はベクターからなるためテクスチャを必要としません。詳しくは以下を参照してください。
