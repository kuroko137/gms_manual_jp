テクスチャグループ

テクスチャグループマネージャーは IDE のツールメニューから開くことができ、テクスチャグループの追加、削除、名前変更を行うことが可能です。GameMaker Studio 2 ではそれぞれのグラフィックアセット（スプライト、タイルセット、フォント）にテクスチャグループを割り当て、ゲーム中にハードウェアが実行するテクスチャスワップ（切り替え）の回数を最適化できます。利用するには、このウィンドウから必要なテクスチャグループを定義し、グラフィックアセットをそれぞれ開いてドロップダウンメニューからグループを指定する必要があります。または、アセットブラウザの右クリックメニューを開いてアセットをテクスチャグループに追加しましょう。複数のアセット（またはグループフォルダ）を選択してから右クリックメニューを開けば、アセットをまとめてグループに追加できます。以下の画像では複数のタイルセットを追加しようとしています。

覚えておきたいことは、テクスチャグループには複数のテクスチャページを追加でき、テクスチャグループとページは異なるものだということです。もしレベル（ステージ）ごとに専用のグラフィックが多く用意されている場合、テクスチャグループはレベルごとに割り当てることになるでしょうが、画像にはグループごとに 2 つ以上のテクスチャページが必要となります。

デフォルトではゲームの画像すべてが、常に存在する 「Default」のテクスチャグループに追加されますが、新規追加ボタンをクリックすれば新しいテクスチャグループを作成できます。また、グループの名前を選択すると適切な名前を入れ直すことができます。グループを作成した直後は、スプライト、タイルセット、フォントの部分に「このグループは空です」というメッセージが表示されますので、それぞれのグラフィックアセットを開いて追加しなくてはなりません。一番下のリソースを追加ボタンをクリックしてリソースを追加するか、画像、タイルセット、フォントそれぞれのエディタからテクスチャグループを選択しましょう。Default 以外のテクスチャグループはいつでも削除可能で、削除するとグループ内のリソースはすべて Default グループに移されます（もし間違ってグループを削除してしまった場合は、 / + "Z" で取り消すことができます）。

テクスチャグループを定義してからグラフィックアセットを割り当てると、テクスチャグループエディタのメインウィンドウ左側に一覧表示されます。そのうちのスプライトをクリックすると右側にプレビューウィンドウが開きます。大規模なプロジェクトでどの画像がグループに割り当てられているか確認したい場合に便利でしょう。割り当てられている画像に複数のサブ画像がある（アニメーションする）場合、プレビューウィンドウには最初のサブ画像のみ表示されます。また、リソースを右クリックするとメニューが開き、別のテクスチャグループに移すか、アセットをエディタで開くことができます。

なお、リストにはスプライト、タイルセット、フォントといったアセットがそれぞれ別々のものとして表示されていますが、実際にはすべて同じテクスチャページに追加されることになり、リソースの種類ごとにページが分けられることはありません。ただし 1 つだけ例外があり、それはテクスチャページを分けるがチェックされているスプライトアセットです。この場合は、そのスプライト専用のテクスチャページが用意されます（詳しくはスプライトエディタの「テクスチャ設定」の節を参照してください）。

For more information on texture pages and how they are generated, please see the following page:

  • Texture Pages

The group settings in the Texture Group Editor are used to determine exactly how the texture pages for a given group should be created, with the following options available:

  • Allow Scaling - Un-flagging this will force GameMaker Studio 2 to not scale the texture group under any circumstances. This can be useful, especially when dealing with universal apps, as it allows you to put specific resources like fonts in a group of its own to be used when the app is run on a specific device. For example, fonts can look great on a retina display, but when they are scaled down to draw on a normal display they can look terrible so you can assign them to a special non-scaling group - if developing for the iPhone, you may wish to have this checked, as apps have been rejected due to the images being blurred by texture scaling. This is flagged as on by default.
  • Automatically Crop - When you add images into GameMaker Studio 2 , if they have any "space" around them (ie 100% transparency) this is automatically cropped when it is added into the texture page and "Automatically Crop" is flagged. However, if, for example, you are using the image as a non-repeating texture for primitives or anything similar, this means that the UV coordinates for texture mapping are not going to be correctly represented. In these cases you can un-flag this option and the sprite will be added to the texture page "as is", maintaining any transparencies that it may have around the edges. This is flagged as on by default.
  • Generate Mipmaps - Flagging this option (it is off by default) will tell GameMaker Studio 2 that the original texture page is to be scaled and filtered into multiple resolutions within the texture file. This is normally off, but if you are using cameras to zoom in or out on large areas of the game room, or you are working with a 3D game, then you may want to enable this and then use the Mipmapping Functions to adjust how the game deals with the different mipmap levels. When you flag this check box the "Border Size" value will be set to 8 (if it was set to less than 8 originally). This is because, when using mipmaps, you need larger borders between images on the texture pages otherwise they start bleeding together at higher mip levels. The default value of 8 will allow the image to be reduced to about an eighth of its original size before the sub-images start bleeding together, but you can change this to any value that is required to suit your needs. Note that if you are using the "Separate Texture Page" option for sprites, then you do not need to go through each texture here and flag it to generate mipmaps, but instead you can go to the General Game Options and flag the option "Generate mipmaps for separate texture pages".
  • Border Size - The border size will change the overlap (or "pixel doubling") amount around the edges of all assets on the pages generated for the texture group so that when the images are scaled, there are no gaps (or seams) visible between them. The default is 2 pixels, but in extreme cases where you are zooming the view in or out a great deal then you may need to increase this value. Note that the Tile Set Editor has an option to set the output border for tiles (and also provides an explanation of why this is necessary), and this is separate to the border setting here - basically, tiles are generated with an output border for each tile on a temporary texture, and this is then added to the texture page with the border given here around the outside edge. Also note that enabling Mipmaps will automatically set the default border size for the texture page to 8 to help prevent "bleeding" between textures at higher mip levels, but this can be changed.
  • Parent Texture Group - Here you can choose any of the existing groups to be the "parent" group for the currently selected one. "Child" groups will inherit any changes made to the parent texture group and are primarily for resource management when working with cross platform games. A texture group with a parent is still a texture group in its own right and will generate texture pages the same as any other group, but assigning them a parent allows you to exclude large chunks for different platforms. For example, you can add a whole load of Android player sprites to a "player" group, and Android enemy sprites to an "enemy" group, then make them both use an "android" parent group. If you then exclude the "android" parent group from the Windows or other platform build, the child groups will be excluded too.

Other than those settings you can also choose to export (or not) any given texture group to any given platform by simply checking or un-checking the appropriate boxes from the Use Group On... section of the window. Note though that the <Default> texture group is always exported to every platform, and you can't check/un-check any of them (only custom texture groups permit this).

Texture Groups are also linked to the Configurations settings. What this means is that you can define texture groups for each individual configuration of your game, and then set the sprite, tile set or font resource assignments to these different groups and it will be "remembered" when you change to a different configuration. The same will happen to any export options that you have chosen in the Texture Group window, so you can set the texture groups to export to specific platforms on a per-configuration basis.

One use for this is, for example, that you could make an iPad configuration and for that create 2048x2048px texture pages (setting this up via the Game Options). You would then assign all the graphics assets to the appropriate texture group, and when finished, you would then change to a different configuration (for HTML5, for example) and from the Texture Group window, create new texture groups that are 512x512px, assigning the assets to these new groups. There is also a button at the bottom of the page that permits you to copy texture group settings from one configuration to another.

Finally, texture groups are only relevant to bitmap images, and non-bitmap sprites will not be assigned to texture groups. Some of them will already have their own unique texture page (skeleton animation sprites) while others just don't require textures as they are comprised of vector.
