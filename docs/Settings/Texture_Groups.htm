テクスチャグループ

テクスチャグループマネージャーは IDE のツールメニューから開くことができ、テクスチャグループの追加、削除、名前変更ができます。GameMaker Studio 2 では画像アセット（スプライト、タイルセット、フォント）にテクスチャグループを割り当て、ハードウェアによって任意のタイミングで実行されるゲーム中のテクスチャスワップ（切り替え）回数を最適化することが可能です。このように機能させるには、まずマネージャーの「新規追加」ボタンをクリックして必要なテクスチャグループを定義し、それから各画像アセットを開いてドロップダウンメニューから所属グループを指定しなくてはなりません。また、アセットブラウザの右クリックメニューからテクスチャグループに追加することも可能で、複数のアセット（またはグループフォルダ）を選択してからメニューを開いた場合、アセットをまとめてテクスチャグループに追加できます。以下の画像では、タイルセットをまとめて追加しようとしています。

テクスチャグループには複数のテクスチャページを追加できますが、テクスチャグループとページが異なるものだということは覚えておきましょう。たとえばレベル（ステージ）ごとに専用の画像が多数ある場合、テクスチャグループをレベルごとに割り当てれば良いのですが、各画像には最低でも 2 つのテクスチャページがグループ単位で必要となります。

ゲームのあらゆる画像は、デフォルトでは必ず存在する "Default" テクスチャグループに追加されますが、新規追加ボタンをクリックすると新しいテクスチャグループを作成できます（名前の部分を選ぶことで好きな名前を再入力できます）。グループを作成した直後はスプライト、タイルセット、フォント部分に「このグループは空です」というメッセージが表示されることになり、各画像アセットを開いて追加しなくてはなりません。一番下のリソースを追加ボタンをクリックしてリソースを追加するか、画像、タイルセット、フォントの各エディタからテクスチャグループを選択しましょう。Default 以外のテクスチャグループは自由に削除可能で、削除するとグループ内のリソースはすべて Default グループに移されます（間違ってグループを削除してしまった場合は、 / + "Z" で取り消すことができます）。

テクスチャグループを定義して画像アセットを割り当てると、テクスチャグループエディタの左側にアセットが一覧表示され、そのうちのスプライトをクリックした場合は右側にプレビューが表示されます。大規模なプロジェクトでグループに割り当てられている画像を確認したいときに役立つでしょう。割り当てられている画像に複数のサブ画像がある（アニメーションする）場合、プレビューウィンドウには最初のサブ画像のみ表示されるためご注意ください。また、リソースを右クリックするとメニューが表示され、別のテクスチャグループに移したり、対応するエディタで開くことができます。

ここではスプライト、タイルセット、フォントといったアセットがそれぞれ別々のものとして並べられていますが、実際にはすべて同じテクスチャページに追加されることになり、リソースの種類ごとにページが分けられることはありません。ただし「テクスチャページを分ける」としてチェックされているスプライトアセットは例外となり、そのスプライト専用のテクスチャページが用意されます（詳しくはスプライトエディタのページから「テクスチャ設定」を参照してください）。

テクスチャページの役割や、どのように生成されるかについては以下のページで詳しく解説されています。

  • テクスチャページ

テクスチャグループエディタのグループ設定では、テクスチャページの作成方法を以下のオプションによって正確に決めることができます。

  • 拡大縮小の許可 - チェックを外すと、いかなる場面でもテクスチャグループに拡大縮小が適用されなくなります。ユニバーサルアプリ（複数のデバイスに対応するアプリケーション）を扱うにあたって特に役立つオプションで、フォントなどの一部リソースを専用のグループに入れ、アプリを特定のデバイスで実行したときに利用することができます。たとえば Retina ディスプレイでは綺麗に表示されるものの、通常のディスプレイでは見た目が悪くなってしまうフォントがある場合、拡大縮小しない専用のグループに割り当てるといったような使い方です。もっとも iPhone 向けに開発している場合は、テクスチャの拡大縮小によって画像がぼやけたせいでリジェクトされたアプリがあるためチェックを入れておくべきでしょう。デフォルトでは有効としてチェックされています。
  • 自動クロップ - GameMaker Studio 2 に画像が追加されたとき、画像のまわりに余白（完全な透明部分）があり、なおかつ「自動クロップ」がチェックされている場合はその余白を自動的に切り取ります。ただし、画像をプリミティブなどを対象とする繰り返し無効テクスチャとして扱っている場合、テクスチャマッピングの UV 座標が正確に再現されないことになります。このような場合では、チェックを外すことでスプライトを元のままテクスチャページに追加でき、端まわりの透明部分を残すことができます。デフォルトでは有効としてチェックされています。
  • ミップマップを生成 - デフォルトでは無効ですが、チェックすると元のテクスチャページがテクスチャファイル内で複数の解像度に拡大縮小、フィルタリングされるようになります。通常は無効にしますが、ルームの広い範囲でカメラをズーム操作したり、あるいは 3D のゲームを制作しているなら有効に切り替え、各ミップマップレベル（段階）をゲームでどのように処理するかをミップマップ関数によって調整するといいでしょう。チェックすると境界のサイズが 8 に変更されますが（元から 8 未満の場合）、これはミップマップ使用時、テクスチャページの画像間により広い境界が必要となるためであり、サイズが小さいとミップマップレベルが高くなるにつれ「にじみ」が生じてしまいます。デフォルト値の 8 なら、サブ画像がにじみ始めるまでに画像を元のサイズの約 1/8 まで縮小できることになりますが、この値は必要に応じて変更可能です。なお、スプライトの「テクスチャページを分ける」オプションを利用している場合、各テクスチャにチェックを入れてミップマップを生成させる必要はなく、一般ゲームオプションを開いて「分けたテクスチャページのミップマップを生成」を有効にするといいでしょう。
  • 境界のサイズ - テクスチャグループのページ生成時、ページに並べられる各アセットの端部分の重複（もといピクセル倍増）量となるもので、変更すると画像を拡大縮小してもすき間（あるいはつなぎ目）が間に表示されなくなります。デフォルトでは 2 ピクセルですが、ビューのズーム倍率を極端に変更するような場合では値を増やさなくてはなりません。なお、タイルセットエディタにはタイルの出力境界を設定できるオプションがありますが（リンク先でその必要性についても説明されています）、ここでの境界設定とは異なるものです。具体的に説明すると、タイルはそれぞれのタイルの出力境界を反映して一時テクスチャに生成され、それからここで指定した外側部分の境界を反映してテクスチャページに追加されます。また、ミップマップを有効にするとテクスチャページのデフォルト境界サイズが自動的に 8 となり、ミップレベルが高いときの「にじみ」を防ぐことができますが、別の値に変更することも可能です。
  • 親テクスチャグループ - 現在のグループの「親」となる既存のグループを指定します。「子」のグループは親のテクスチャグループに加えられた変更をすべて引き継ぐため、主にクロスプラットフォームのゲームを扱う場合のリソース管理目的で利用します。親があっても子のテクスチャグループは独立しており、他のグループと同じようにテクスチャページを生成しますが、親を割り当てることでプラットフォームごとにテクスチャグループをまとめて除外できます。たとえば "player" グループに Android 用のプレイヤースプライト、"enemy" グループに Android 用の敵スプライトを大量に追加し、両方に "android" という親グループを割り当てたとしましょう。この場合、Windows や他プラットフォームのビルドから "android" グループを除外すれば、子グループも同じように除外されます。

これらの設定以外にも、グループの使用先に表示されている各チェックボックスからテクスチャグループの出力プラットフォームを切り替えることができます。ただし<Default>テクスチャグループはすべてのプラットフォームに必ず出力されるため、チェックを付け外しすることはできません（カスタムテクスチャグループのみ可）。

テクスチャグループは構成とも関連しています。具体的には、構成別にテクスチャグループを定義し、スプライト、タイルセット、フォントリソースをそれらのグループに割り当てた場合、構成を変えても設定がそのまま記憶されるということです。テクスチャグループのウィンドウ内にある出力オプションも同じであり、構成ごとにテクスチャグループの出力プラットフォームを設定できます。

使用例として、iPad の構成で 2048x2048px のテクスチャページを作成するとしましょう（ゲームオプションから設定を行います）。あとは画像アセットを適切なテクスチャグループに割り当てたのち、たとえば HTML5 に構成を切り替え、テクスチャグループエディタから 512x512px のテクスチャグループを作成してアセットを割り当てるといった流れになります。

なお、テクスチャグループはビットマップ画像のみに関連するもので、非ビットマップスプライトはテクスチャグループに割り当てることができません。スケルタルアニメーション・スプライトのように、あらかじめ専用のテクスチャページが用意されるスプライトもありますが、それ以外のスプライトはベクターによって成り立つためテクスチャを必要としません。詳しくは以下を参照してください。
