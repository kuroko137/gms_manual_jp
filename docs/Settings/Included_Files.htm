インクルードファイル

GameMaker Studio 2 にはプログラム内から作成したアセットだけではなく、ゲーム実行中に利用できるインクルードファイルを追加できます。どのようなファイルも追加できますが、標準の*.ini ファイル、スプライト、背景画像、言語テキストファイルといったものがよく利用されています。

インクルードファイルは、アセットブラウザの追加メニューからエディタを開いて追加します。

メニューの項目を選ぶと、このようにインクルードファイルエディタが開きます。

リソースに含めるファイルを追加するには、エクスプローラーで開くボタンをクリックしましょう。するとファイルエクスプローラーによってプロジェクトの datafiles フォルダが開かれます。あとは必要なディレクトリやファイルを追加すれば、エディタが自動的に更新されるでしょう。

なお、名前の変更、ロケーションの整理といった編集はすべて標準のファイルエクスプローラー上で行わなくてはなりません。

エディタにファイルが追加されると、ダブルクリックしてファイルプロパティを開けるようになります。

このウィンドウにはファイルの名前、ディスク上でのサイズ、ゲームのコンパイル / テスト時にエクスポートされるプラットフォーム一覧が表示されます。「すべて」または「無効」ボタンをクリックするか、一覧のチェックを切り替えることで出力するプラットフォームを切り替えることができます。

インクルードファイルがゲームにどう保存され、どのように利用できるかについては、ファイルシステムのページで詳しく解説されています。

重要！サウンドファイルを追加する場合、各ストリーミングサウンドアセットと異なる名前にしないといけません。たとえば "main_music" というサウンドアセットがあるのに "main_music.ogg" というインクルードファイルが追加されると、プロジェクト実行時に未定義となります。これはストリーミングファイルもメインプロジェクトのバンドル外に保存されるためです。サウンドファイルを追加する際はご注意ください。

インクルードファイルには DLL （ Windows 上）や DyLib （ macOS 上）を含めることも可能で、external_define()によって独自の関数を作成できます。こうして作成した関数は external_call()によって呼び出せます。

警告！インクルードファイルから関数を作成した場合、拡張機能システムとは互換性がなくなります。拡張機能には定数や関数を追加するためのメソッドがすでに用意されているため、拡張機能に追加したファイルを external_*()関数で扱うことはできません。
